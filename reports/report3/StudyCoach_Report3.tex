The aim in this report is to analyze the outputs of the approaches collected in Report 2,
and try out modifications to see how they affect performance on intrinsic metrics
and qualitatively. Our task is incongruence detection: given a scientific figure and
a student answer, a model must (1) classify the verdict (correct / partially correct
/ incorrect), (2) identify the error category (factual, conceptual, omission), and
(3) generate coaching feedback.

\section{Analysis (4 pages)}

\subsection{\points{2} Intrinsic Metrics}

For a VLM to grade student answers about scientific figures, it needs three core
skills: recognizing incomplete answers, knowing when coaching is required, and
extracting specific values from visual data. We measure these independently of
end-to-end verdict accuracy using per-example outputs from the 8B evaluation set.

SPIQA+ contains 174 examples with three verdict classes: 66 correct (37.9\%),
61 partially correct (35.1\%), and 47 incorrect (27.0\%). Since coaching is only
needed for non-correct answers, we compute intrinsic metrics on the 108 examples
where ground truth is partially correct or incorrect.

\paragraph{Intrinsic Metric 1: Partially-Correct Recall (PC Recall).}

Can the model recognize that a student answer is \textit{incomplete} rather than
fully wrong or fully right? Study Coach's core coaching scenario involves answers
that are partially correct---the student understood something but missed something
else. An educator who cannot reliably identify this middle class cannot target
feedback appropriately.

We define PC Recall as the fraction of ground-truth Partially Correct examples
correctly classified:
\begin{equation}
\text{PC Recall} = \frac{\text{TP}_\text{PC}}{\text{TP}_\text{PC} + \text{FN}_\text{PC}}
\end{equation}

This is not overall verdict accuracy: a model can maintain moderate accuracy by
collapsing partially correct answers into the incorrect class, generating feedback
of the wrong kind. PC Recall isolates the hardest, most coaching-relevant class
and is computed on the 61 ground-truth partially correct examples only.

At 8B scale, PC Recall drops sharply as visual context is added: text-only
44.3\%~$\to$~multimodal 19.7\% (Table~\ref{tab:intrinsic}). The model correctly
identifies fewer than 1 in 5 partially correct answers in the multimodal
condition---the same condition that produces the best coaching feedback. This
sharpens the verdict--feedback dissociation from Report 2 into a precise, per-class
finding: visual input specifically degrades recognition of the middle class that
most needs coaching.

\paragraph{Intrinsic Metric 2: Feedback Suppression Rate (FSR).}

Does the model generate coaching when it is needed? When the model predicts
``Correct,'' it outputs N/A feedback by prompt design. Suppressing feedback on
a truly correct answer is appropriate; suppressing it on an incorrect or partially
correct answer silences the coaching pipeline when the student needs help. We define
FSR on the 108 non-correct examples only:
\begin{equation}
\text{FSR} = \frac{\text{\# Correct predictions on non-correct GT}}{\text{\# non-correct GT examples (}N{=}108\text{)}}
\end{equation}

FSR measures a prerequisite skill: recognizing that coaching is required at all.
A model with high FSR produces false-positive verdicts with no explanation,
directly harming the student. FSR is distinct from verdict accuracy---a model
could classify incorrect answers correctly while still suppressing feedback on
every partial-credit case.

FSR increases substantially with visual context: text-only 13.9\%~$\to$~vision-only
30.6\%~$\to$~multimodal 28.7\% (Table~\ref{tab:intrinsic}). Adding visual input
causes the model to incorrectly accept more student answers as correct,
particularly for omission errors where the answer is plausible but structurally
incomplete. In the text-only condition, the model inappropriately suppresses
feedback on roughly 1 in 7 non-correct examples; in visual conditions, this rises
to nearly 1 in 3.

\paragraph{Intrinsic Metric 3: Visual Grounding Score (VGS).}

Can the model extract specific values from figures and reference them in feedback?
All three error types benefit from visual grounding (Report 2, H3): factual errors
require verifying specific numbers (``The figure shows 84.6\%, not 89\%''),
conceptual errors benefit from citing values that reveal patterns (``X increased
from 10 to 50''), and omission errors---which show the \textit{largest} feedback
gains (+33pp)---require the figure to reveal what the student failed to mention.
We define VGS on all 108 non-correct examples, excluding suppressed feedbacks:
\begin{equation}
\text{VGS} = \frac{\text{\# feedbacks citing specific values from figure}}{\text{\# non-suppressed feedbacks (}N{=}108\text{)}}
\end{equation}

VGS measures whether visual information is actually translated into text during
reasoning, independently of whether the verdict is correct. It is conceptually
related to Qwen3-VL's square-root reweighting loss, which scales multimodal
sample loss by $\sqrt{N_{\text{text}}/N_{\text{multimodal}}}$ to preserve
visual grounding during pre-training without degrading text capabilities.
VGS captures whether this grounding carries through to inference on our task.

VGS increases substantially with visual input: text-only 35.5\%~$\to$~vision-only
52.0\%~$\to$~multimodal 49.4\% at 8B (Table~\ref{tab:intrinsic}). At 32B, the
pattern strengthens: C1 37.2\%~$\to$~C4 56.0\% ($\Delta{=}+$18.8pp). This confirms
that the model \textit{does} use visual information for explanation generation even
when it cannot use it for classification---providing intrinsic evidence for the
verdict--feedback dissociation. Omission errors show the largest VGS gains
(+19.2pp at 8B), consistent with Report 2's finding that the figure reveals what
the student failed to mention.

\paragraph{Evaluation methodology: LLM-as-Judge for feedback quality.}

Because open-ended coaching feedback cannot be evaluated with n-gram metrics alone
(F1, ROUGE-L, and BLEU show near-zero variation across conditions at both scales;
see Report 2, Table~10), we use Claude Sonnet~4 as an automated judge. Each predicted
feedback is classified as \textbf{Match} (correctly identifies error and explains
why), \textbf{Partial} (right direction but imprecise or incomplete), or
\textbf{Unmatched} (does not align with ground truth). Full definitions are in Report 2.

We report \textbf{Match\%} (strict: exact matches only) and \textbf{Soft Match\%}
(Match + Partial). At 32B, C4 achieves 20\% strict match but 58\% soft
match---meaning 38\% of responses are approximately correct but not precise.
Human evaluation ($n{=}10$/condition, 8B) confirmed the pattern: human match
ranged from 40\% (C1) to 80\% (C4), validating soft match as the practical metric
for coaching. We use strict match (Match\%) in Table~\ref{tab:intrinsic} since it
separates conditions more cleanly.

\noindent\textit{Routing (Modification 1):}
Routing is a post-hoc combination of existing C1 and C4 results that demonstrates the potential of condition-based task routing. Since C1 (text-only) excels at verdict classification while C4 (multimodal) excels at feedback generation, Routing combines: (1) verdict, PC Recall, and FSR from C1, and (2) Match\%, SoftM\%, and VGS from C4. This is not a separate inference run but shows the upper bound achievable by a two-stage pipeline that routes verdict to C1 and feedback generation to C4.

\textit{Key observations (8B):}
\begin{itemize}[itemsep=1pt, topsep=2pt, leftmargin=*]
  \item \textbf{PC Recall drops with visual input:} C1 (44.3\%) $\to$ C4 (19.7\%)---visual information introduces noise into classification boundary
  \item \textbf{VGS rises with visual input:} C1 (35.5\%) $\to$ C4 (49.4\%)---model extracts more values from figures into feedback
  \item \textbf{Same visual input, opposite effects:} Hurts classification ($-$8pp verdict) but helps explanation ($+$4pp Match)
  \item \textbf{Routing recovers both:} C1 verdict (56\%, 44.3\% PC Recall) + C4 feedback (6\% Match, 49.4\% VGS)
\end{itemize}

\textit{Key observations (32B):}
\begin{itemize}[itemsep=1pt, topsep=2pt, leftmargin=*]
  \item \textbf{PC Recall drops with visual input:} C1 (41.0\%) $\to$ C4 (27.9\%)---similar pattern to 8B but smaller drop ($-$13pp vs $-$25pp)
  \item \textbf{VGS rises with visual input:} C1 (37.2\%) $\to$ C4 (56.0\%)---scaling improves grounding ($+$19pp vs $+$14pp at 8B)

\end{itemize}

\begin{itemize}[itemsep=1pt, topsep=2pt, leftmargin=*]

\subsection{Modifications and their Effects}
\points{2}

We implement five modifications motivated by the qualitative failure analysis in
Section~\ref{sec:qual} and the intrinsic metric patterns above. Modification~1
(Condition Routing) is a zero-cost inference-time fix computable from existing Report 2
outputs. Modification~2 is an architectural ablation that tests whether the bolt-on
ViT design of Qwen3-VL causes the verdict--feedback dissociation, by replacing it
with an early-fusion architecture while holding the prompt and evaluation pipeline constant.
Modification~3 (Feedback-Only Prompt) decouples coaching feedback from verdict classification,
testing whether task simplification improves explanation quality.
Modification~4 (Visual CoT) tests whether forcing explicit visual reasoning before the
verdict can improve classification without sacrificing feedback quality.
Modification~5 (MoLoRA) explores figure-type-aware adaptation using mixture of LoRA experts.

\paragraph{Modification 1: Condition Routing (C1 verdict + C4 feedback).}

\textbf{Motivation.} The intrinsic metrics make the verdict--feedback dissociation
precise: PC Recall drops from 44.3\% to 19.7\% as visual context is added (C1 to
C4), while VGS rises from 35.5\% to 49.4\%. These are not correlated failures of
a single configuration---they measure different sub-skills that are genuinely in
tension at 8B scale. The model can use visual information for extraction (improving
VGS and feedback quality) but not for classification (degrading PC Recall). Routing
assigns each sub-task to the condition best suited to it: C1 for verdict and error
category, C4 for coaching feedback.

\textbf{Implementation.} No new inference is required. We pair per-example C1
predictions (verdict, error category) with per-example C4 predictions (feedback)
using matched (\texttt{paper\_id}, \texttt{question}) keys across the H3/H4
evaluation set. All 108 examples matched successfully (100\% coverage).

\textbf{Results.} Routing achieves the best combined profile across all conditions:
verdict 56.0\% and PC Recall 44.3\% (matching C1), VGS 49.4\% and Fdbk Match
6.0\% (matching C4), with FSR at 28.7\%. The routing system recovers C1's verdict
precision and C4's visual grounding simultaneously at zero additional inference
cost. The FSR limitation (28.7\%) arises because N/A feedback comes from C4 when
C4 predicts Correct; a production system would override this by using the C1
verdict to force feedback generation regardless of C4's verdict.

\paragraph{Modification 2: Architectural Ablation --- Qwen3.5-9B Early Fusion vs. Qwen3-VL-8B Bolt-On ViT.}

\textbf{Motivation.} The intrinsic metrics reveal a structural paradox: VGS rises
from 35.5\% (C1) to 49.4\% (C4), confirming that visual information is being
extracted and used for explanation generation---yet PC Recall simultaneously
collapses from 44.3\% to 19.7\%, and FSR roughly doubles. The model \emph{can}
use figure content; it just cannot apply it to the verdict classification decision.

One architectural explanation is the \emph{bolt-on} design of Qwen3-VL.
Per the Qwen3-VL technical report~\cite{bai2025qwen3vl}, the model uses a three-module pipeline: a
SigLIP-2 SO-400M vision encoder (trained separately from the LLM), a two-layer
MLP merger augmented by DeepStack residual connections (which inject features
from three intermediate ViT layers into the first three LLM layers), and a
Qwen3-8B/32B decoder backbone. The modular separation is confirmed by the
pre-training recipe: Stage~0 freezes both the vision encoder and the LLM
backbone while training \emph{only the merger}, establishing a clear
representational boundary between the two modalities. DeepStack improves
multi-level visual feature injection but does not eliminate this boundary;
the LLM backbone was pre-trained as a text model and receives visual tokens
as an external input stream rather than being co-trained with visual objectives
from the start. This late-injection design may explain why the model can use
injected visual features for explanation generation (improving VGS and feedback
quality) but cannot reliably route those features into the verdict decision
boundary (degrading PC Recall).

Qwen3.5-9B~\cite{qwen2026qwen35} uses a fundamentally different design: \emph{early fusion}. Rather
than a three-module pipeline (separate SigLIP-2 encoder $\to$ MLP merger
$\to$ LLM), Qwen3.5-9B co-trains vision and language from the start of
pre-training within a unified Gated DeltaNet hybrid architecture---no separate
vision encoder, no MLP merger, no Stage~0 alignment phase. Visual and textual
representations occupy the same token space throughout all 32 layers from the
first training step. This is not a cosmetic difference: the model was never in
a regime where visual tokens arrived as an external stream bolted onto a
text-trained LLM backbone. If the three-module bolt-on design of Qwen3-VL
(SigLIP-2 encoder + MLP/DeepStack merger + frozen-then-thawed Qwen3 decoder)
causes the PC Recall collapse observed under visual input, Qwen3.5-9B should
show a smaller or reversed verdict gap (C4 accuracy $\geq$ C1 accuracy) at
comparable parameter count ($\sim$9B).

\textbf{Methodology.} We ran the identical evaluation pipeline from Report 2
(same prompts, same parsing logic, same metrics) on Qwen3.5-9B for conditions
C1 (text-only) and C4 (multimodal), N=108 examples. The only change was the
model ID. Thinking mode was disabled to preserve structured output format.

\textbf{Results.} The hypothesis is \textbf{confirmed}: Qwen3.5-9B shows a
\emph{reversed} verdict gap compared to Qwen3-VL-8B. Key findings:

\begin{itemize}[itemsep=1pt, topsep=2pt, leftmargin=*]
  \item \textbf{Verdict gap reversed:} C1 (38.0\%) $<$ C4 (44.4\%), a $+$6.4pp
    improvement with visual input. Compare to Qwen3-VL-8B where visual input
    \emph{hurt}: C1 (56.0\%) $>$ C4 (48.0\%), a $-$8pp drop.
  \item \textbf{PC Recall stable:} C1 (32.8\%) $\to$ C4 (31.1\%), only $-$1.7pp.
    Compare to Qwen3-VL-8B's collapse: 44.3\% $\to$ 19.7\% ($-$24.6pp).
  \item \textbf{FSR improves with visual input:} C1 (38.9\%) $\to$ C4 (24.1\%),
    a $-$14.8pp reduction (lower is better). This is the \emph{opposite} of
    Qwen3-VL-8B where visual input increased suppression ($+$14.8pp).
  \item \textbf{VGS increases:} C1 (30.3\%) $\to$ C4 (42.7\%), $+$12.4pp---similar
    pattern to Qwen3-VL-8B ($+$13.9pp).
  \item \textbf{Feedback quality dramatically improved:} Match\% jumps from
    15.7\% (C1) to 41.7\% (C4), a $+$26.0pp gain. Compare to Qwen3-VL-8B's
    modest $+$4.0pp gain (2\% $\to$ 6\%). The 9B early-fusion model achieves
    \textbf{7$\times$ higher C4 Match\%} than the 8B bolt-on model.
\end{itemize}

\textbf{Interpretation:} The early fusion architecture resolves the
verdict--feedback dissociation observed in Qwen3-VL-8B. Visual input now helps
\emph{both} classification and explanation generation simultaneously. The lower
absolute verdict accuracy (38--44\% vs 48--56\%) reflects the 9B model's smaller
capacity, not an architectural flaw---this can be addressed through fine-tuning
on SPIQA+.

\textbf{Implication for Report 4:} Per the decision matrix, since C4 verdict
accuracy $>$ C1 for Qwen3.5-9B, \textbf{early fusion at 9B is sufficient}. The
Report 4 fine-tuning backbone should be Qwen3.5-9B (or the 27B dense variant for
higher absolute accuracy), not Qwen3-VL.

architectural or scale-dependent. The decision matrix was:

\paragraph{Modification 3: Decoupling Feedback Generation from Classification.}

\textbf{Motivation.} A central finding from Report~2 was a verdict--feedback dissociation (H2): the multimodal condition (C4) produced the best coaching feedback but the \emph{worst} verdict accuracy across both model scales. Specifically, text-only C1 achieved 56\% verdict accuracy at both 8B and 32B, while C4 achieved only 48\% (8B) and 54\% (32B). Meanwhile, C4 led all conditions on feedback quality.

This observation pointed to a structural problem: our prompt asked the model to simultaneously produce three outputs---a verdict, an error category, and free-text coaching feedback---in a single pass. Under this multi-task framing, the classification objective and the explanation objective compete for the model's capacity. We simplified the prompt to a single objective: \textbf{generate coaching feedback only}, with no reference answer and no verdict or error category requested. This isolates the multimodal grounding capability for feedback generation.

\textbf{Experimental Setup.} We applied the feedback-only prompt to the multimodal condition (C4) on both Qwen3-VL-8B-Instruct and Qwen3-VL-32B-Instruct ($N{=}50$ each), judged by Claude Sonnet 4.

These results provide empirical motivation for the condition-routing architecture (Modification~1): use text-only (C1) for grading and a feedback-focused multimodal prompt for coaching.

\paragraph{Modification 4: Visual Chain-of-Thought Prompting (C4-CoT).}

\textbf{Motivation.} The qualitative analysis in Section~\ref{sec:qual} identifies
the root cause of both verdict inflation and hallucinated correction: without
structured reasoning over the figure, the model falls back on plausibility assessment
rather than evidence-based judgment. All baseline evaluations use \texttt{/no\_think}
to suppress the model's native chain-of-thought, meaning the model produces a
verdict without any explicit intermediate reasoning about figure content.
Modification~4 tests whether forcing the model to describe what it sees in the figure
\textit{before} judging can improve verdict accuracy---particularly PC Recall---without
sacrificing feedback quality.

\textbf{Implementation.} We create a modified system prompt
(\texttt{incongruence\_eval\_v1\_cot.txt}) that inserts a visual analysis step
before the verdict. The model must: (1)~identify key data points, values, or patterns
visible in the figure, (2)~note specific numbers, trends, or comparisons relevant to
the question, and (3)~compare the student's claims against what the figure shows. The
output format adds a \texttt{Visual Analysis} field before the existing
Verdict/Error~Category/Feedback fields. The \texttt{/no\_think} directive is
retained---the CoT is visible in the structured output, not in a hidden reasoning
block. This isolates the effect of structured visual reasoning from the model's
internal thinking mode.

We run C4-CoT on all $N{=}108$ non-correct examples using Qwen3-VL-32B, the same
model and evaluation set as the C4 baseline. The only variable is the system prompt;
all other parameters (model endpoint, temperature, user message format) are identical.

The most striking finding is that CoT \textbf{substantially reduces feedback
suppression}: FSR drops from 30.6\% to 17.6\% ($-$13.0pp), meaning the model
incorrectly accepts 13 fewer non-correct answers as ``Correct'' out of 108. This
directly addresses the verdict inflation failure mode identified in
Section~\ref{sec:qual}---the visual analysis step forces the model to examine figure
evidence before committing to a verdict, making it harder to dismiss errors without
justification.

However, PC Recall drops from 27.9\% to 23.0\% ($-$4.9pp). The model classifies
fewer partially correct answers correctly, instead over-correcting them to Incorrect.
The visual analysis step helps the model detect \textit{that something is wrong}
(lower FSR) but makes it harder to recognize \textit{how much is wrong} (lower PC
Recall). This produces a net drop in verdict accuracy from 54.0\% to 48.1\%.

VGS remains essentially unchanged (56.0\% $\to$ 55.1\%), confirming that CoT does
not degrade visual grounding in feedback---the model continues to cite specific
figure values at the same rate.

\textbf{Interpretation.} CoT addresses one half of the verdict--feedback
dissociation: it reduces false ``Correct'' predictions (lower FSR) by making the
model confront figure evidence before judging. But it introduces a new failure
mode---\textbf{over-correction}---where the visual analysis step leads the model to
flag discrepancies that are minor enough to warrant ``Partially Correct,'' not
``Incorrect.'' Table~\ref{tab:cot_qual} in Section~\ref{sec:qual} illustrates both
patterns with paired examples. This suggests that the dissociation is not purely a
reasoning depth problem (solvable by more explicit reasoning) but involves
calibration of the three-class decision boundary. A production system could combine
CoT's lower FSR with routing (Mod~1): use C1 for the verdict boundary and C4-CoT
for feedback generation, capturing both reduced suppression and superior
classification.

\paragraph{Modification 5: Mixture of LoRA Experts for Figure-Type-Aware Adaptation}

We use Qwen3-VL-8B as the baseline for this modification. With zero shot inference
on test-A dataset (N=108), we observe following metrics for each figure type.

\begin{table}[h]
  \centering
  \begin{tabular}{lrrr}
    \toprule
    Figure Type & ROUGE-L & METEOR & BERTScore-F1 \\
    \midrule
    Schematic & 0.33 & 0.31 & 0.86 \\
    Table     & 0.31 & 0.30 & 0.85 \\
    Plot      & 0.30 & 0.28 & 0.84 \\
    \bottomrule
  \end{tabular}
  \caption{Evaluation metrics by figure type}
  \label{tab:metrics-by-fig-type}
\end{table}

As per Table \ref{tab:metrics-by-fig-type} there are variations in scores per figure-types (specially 
BERTSCore-F1 where $\sim$\%2 is a meaningful difference from a strong baseline of 0.84) .
This gave us the idea: \textit{can we augment the LLM backbone with a Mixture-Of-Experts tuned on visual inputs, 
with experts specializing on different figure types?}

The details of the fine-tuning task given below.

\textbf{Data}: Training used 8,000 examples sampled from 1,500 papers (SPIQA+ train), with a paper-disjoint validation 
set of 160 examples drawn from a separate 200-paper pool. Evaluation used 108 examples from the SPIQA+ Test-A split. 
We chose 8k training size based on loss curve analysis of a preliminary 10k run, which showed convergence at 
approximately 8,000 examples with negligible improvement beyond that point.

\textbf{Architecture}: Each target linear layer ($q_{proj}$, $v_{proj}$, $o_{proj}$, $gate_{proj}$) on the LLM backbone 
is replaced with a module $MoLoRALinear$ containing K=4 parallel LoRA adapters (rank r=16, $\alpha$ = 32, dropout=0.05) and a 
shared MLP router (Figure \ref{fig:molora-experts-layer}) . The router takes the mean-pooled output of the vision-language projector/merger as input (a visual 
summary vector with dim=4096) and produces K softmax weights via a two-layer MLP (hidden dim=128). These weights are 
applied as a soft weighted combination of the K expert adapter outputs, so all experts participate in every forward
pass (i.e.: soft routing). The vision encoder and merger are frozen; only the LoRA adapter weights and router were
trained.

\begin{figure}
    \centering
    \includegraphics[width=0.8\linewidth]{MoLoRALinear_Layer.jpg}
    \caption{Mixture-of-LoRA-Experts Layer}
    \label{fig:molora-experts-layer}
\end{figure}

\textbf{Router collapse prevention:} We noticed that the structured output format creates strong directional LM loss 
gradients in early training. This causes naive soft routing to collapse to a single dominant expert 
within 20–30 steps (dominant expert ends up having around 80\% of routing weight). We used three
complementary mechanisms to mitigate this: \newline \newline
(1) \textit{Temperature annealing}: We initialize the router softmax temperature T at 10.0 and then anneal it
linearly to 1.0 over the first 30\% of training steps, forcing near-uniform expert utilisation during 
the early phase
\\
(2) \textit{Entropy Bonus}: We add a term $-0.02 \times H(\text{weights})$ to the loss throughout training, directly 
penalising collapsed distributions
\\
(3) \textit{Router Noise}: We also add a Gaussian noise ($\sigma$=0.5) to router logits during training to encourage exploration.

\textbf{Training setup:} Single NVIDIA A40 46GB GPU, bfloat16 precision. Batch size 1, gradient accumulation 8 (effective batch 8), learning rate $2*10^{-4}$ with cosine schedule, 5\% warmup, 1 epoch. Training took approximately 4 hours per run.

\section{\points{3} Insights}

\textbf{The verdict--feedback dissociation is a task decomposition signal, not a
bug to fix.} Our intrinsic metric analysis sharpens what appeared in Report 2 as a
surprising empirical finding into a precise, mechanistically-grounded claim. PC
Recall drops from 44.3\% to 19.7\% as visual input is added, while VGS rises from
35.5\% to 49.4\%. These measure different sub-skills that are genuinely in tension
at 8B scale: visual information improves value extraction into text (VGS) but
introduces noise into the three-class decision boundary, particularly for the
partially correct middle class that requires the most nuanced judgment. The
condition routing modification confirms this: using C1 for verdict and C4 for
feedback simultaneously recovers C1's PC Recall (44.3\%) and C4's feedback quality
(6.0\% match) at no additional inference cost. This reframes the main model
proposal for Report 4: rather than searching for a single configuration or fine-tuning
objective that improves both metrics jointly, Study Coach should be designed as an
explicit two-stage pipeline with task routing---a lightweight verdict classifier
(the Verdict-Classification Head proposed in Report 2, Section 3.4) decoupled from the
feedback generator. The head would be trained directly on the three-class boundary
using C1 or C2 inputs, while the feedback generator continues to use C4.

\textbf{Modification~2 confirms that architecture, not scale, is the root cause.}
The gap between VGS (49.4\% at C4) and PC Recall (19.7\% at C4) in Qwen3-VL-8B had two
competing explanations: (1) architectural---the bolt-on SigLIP-2 encoder creates a
representational boundary that affects classification more than generation; or
(2) scale-dependent---8B parameters are insufficient for reliable three-class
verdict decisions with visual input. Modification~2 resolves this question
decisively: \textbf{Qwen3.5-9B (early fusion) reverses the verdict gap entirely}
(C4 44.4\% $>$ C1 38.0\%), while also achieving 7$\times$ higher feedback Match\%
than Qwen3-VL-8B at C4 (41.7\% vs 6.0\%). The PC Recall drop is minimal ($-$1.7pp
vs $-$24.6pp), and FSR actually \emph{improves} with visual input ($-$14.8pp).
This confirms the architectural explanation: the bolt-on ViT design of Qwen3-VL
is the cause of the verdict--feedback dissociation, not insufficient model capacity.

\textbf{Visual chain-of-thought addresses suppression but not calibration.}
Modification~4 tests whether the verdict--feedback dissociation can be mitigated
by forcing explicit visual reasoning before the verdict. The results are instructive:
CoT reduces FSR by 13pp (from 30.6\% to 17.6\%), confirming that structured figure
analysis prevents the model from dismissing errors as correct without evidence.
But PC Recall simultaneously drops by 4.9pp, revealing that CoT introduces
over-correction---the model identifies discrepancies between student claims and
figure content but cannot calibrate whether those discrepancies warrant ``Partially
Correct'' or ``Incorrect.'' This is a different failure mode from the baseline:
the original C4 model under-detects errors (high FSR), while C4-CoT over-penalizes
them (low PC Recall). Together with the routing result from Modification~1, this
suggests that the optimal Report~4 architecture combines three components:
(1)~C1-based verdict classification (highest PC Recall), (2)~CoT-enhanced C4
feedback generation (lowest FSR, stable VGS), and (3)~the Verdict-Classification
Head to learn the three-class boundary directly rather than relying on prompt-based
calibration.

\textbf{Report 4 strategy is now clear:} Fine-tune Qwen3.5-9B (or the 27B dense
variant for higher absolute accuracy) on SPIQA+. The early fusion architecture
correctly integrates visual information for both classification and generation.
The lower absolute verdict accuracy (38--44\% vs 48--56\%) reflects the 9B model's
smaller capacity relative to the task, which domain-specific fine-tuning will address.
The condition routing result from Modification~1 provides a strong zero-cost baseline
(56\% verdict, 6\% Match at 8B) that fine-tuning should exceed.
