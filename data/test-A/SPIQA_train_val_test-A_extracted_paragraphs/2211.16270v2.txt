End-to-end models are compelling approaches for ASR due to their simplicity in model architecture and training process. A number of end-to-end models have been proposed in the literature, such as  connectionist temporal classification (CTC) [1], neural transducer [2] and  attention-based encoder decoder (AED) models [3, 4].Due to its convenient properties for streaming ASR, the neural transducer has seen increased research interest over the past years [5, 6, 7], resulting in successful deployment in on-device production settings [8]. However, training a transducer model is typically more complex and resource intensive than training CTC or AED models. Recent efforts have focused on improving performance and memory consumption of transducer training [9, 10, 11, 12, 13, 14].

During transducer training, five components are involved, shown in Fig. 1: acoustic encoder, label encoder, joint network, output layer and loss function. The output layer computes a 4D tensor of shape [B,T,U,V]𝐵𝑇𝑈𝑉[B,T,U,V], with batch size B𝐵B, acoustic sequence length T𝑇T, label sequence length U𝑈U, and vocabulary size V𝑉V. Even for moderate settings, the memory size of the 4D output tensor can quickly grow into the 10s to 100s of GBs - exceeding even the largest GPU memory configurations 111V100: up to 32GB, A100/H100: up to 80GB.

To reduce memory consumption, previous work has proposed various optimizations. Computational optimizations use function merging to avoid intermediate results tensors [2, 9, 10], and reuse memory by overwriting activation tensors in-place with gradient tensors [10]. Further memory savings are achieved by methods that reduce the memory size of the 4D output tensor: reduced precision [11], pruning [14], padding removal [10] and batch-splitting [12].

The batch-splitting method in the NVIDIA NeMo toolkit [12] computes the acoustic encoder for the full batch size, and then splits the remainder of the computation into sub-batches. After the gradients for a sub-batch are computed, the occupied memory can be freed before moving on to the next sub-batch. The full batch gradients are obtained by gradient accumulation.

This work extends previous investigations with a full analysis for the time and space complexity of transducer training. To save memory, we propose a sample-wise method that is similar, but not identical to batch-splitting. Our implementation maximizes the amount of batched computation, and minimizes memory consumption in the remainder of the computation. We propose a novel method to dynamically increase the parallelism during sample-wise computation. We provide a set of thorough performance and memory benchmarks, using parameter settings that are typical for real-world training.

The same or similar loss implementations are used in ASR toolkits such as TorchAudio [15], ESPnet [16] and NeMo [12]. From a practical point of view, the time complexity of transducers may not be a critical issue. The computations with highest time complexity, e.g. in the output layer, are highly parallel and ideal targets for specialized tensor processing accelerators (TPU [17], TensorCores [18]). However, space complexity remains a critical issue, as the size of the 4D tensors (𝐳,𝐡,d​𝐡𝐳𝐡𝑑𝐡\mathbf{z},\mathbf{h},d\mathbf{h}) may exceed memory capacity.

This section describes a sample-wise method to make transducer training more memory efficient. Memory is saved by reducing the effective batch size to B=1𝐵1B=1 for select transducer components.

The method is detailed in Algorithm 1. The notation [b]delimited-[]𝑏[b] refers to indexing into position b𝑏b of the batch dimension. A tensor with the star superscript (∗)(^{*}) corresponds to a single training sample, i.e. the tensor is not batched. First, the acoustic and label encoders run in default batched mode, producing batched encodings (lines 1, 2). Then, the computation switches to a sample-wise mode for the joint network, output layer and loss function (lines 4-11). Both activations and gradients are computed with a batch size of 1, saving memory. When the computation for an individual sample is finished, the gradients are accumulated (lines 8-11). Due to the accumulation, the result is identical to batched computation. By deleting the intermediate tensors (𝐳∗,𝐡∗,d​𝐡∗)superscript𝐳superscript𝐡𝑑superscript𝐡(\mathbf{z}^{*},\mathbf{h}^{*},d\mathbf{h}^{*}), memory is freed (line 12). The remaining downstream gradients for the acoustic and label encoder parameters are computed in default batched mode (lines 13, 14).

When generating training batches, samples are typically selected with a bucketing approach. With bucketing, samples of similar acoustic sequence length are batched to reduce wasted computation on padded segments. However, the transducer combines acoustic and label encodings. The variance in label sequence lengths can lead to a significant amount of padding e.g. in the joint network and output layer. The sample-wise computation allows us to remove all padding, fully avoiding wasted computation.

The proposed method partially reduces the batch size to B=1𝐵1B=1, which reduces efficiency on parallel accelerators. However, some of the parallelism can be recovered by running multiple iterations of the sample-wise computation in parallel (lines 4-12). In our TensorFlow implementation, we use the tf.while_loop with a dynamic setting for parallel_iterations. Based on the size of the intermediate tensor h∗superscripth\textbf{h}^{*}, the parallelism is dynamically controlled to stay within user-defined memory limits. For short batches with small (𝐓,𝐔)𝐓𝐔\mathbf{(T,U)}, higher parallelism can be afforded without exceeding memory limits. For longer batches with large (𝐓,𝐔)𝐓𝐔\mathbf{(T,U)}, parallel_iterations will gradually drop to 1.

The NVIDIA NeMo toolkit proposes a similar batch-splitting method to save memory with neural transducers [12]. The computation of the label encoder, joint network, output layer and loss is split into sub-batches with reduced batch size >=1absent1>=1. The sub-batch size is fixed, and only the acoustic encoder runs in batched mode.In contrast, our proposed method runs both acoustic and label encoders in batched mode, increasing parallelism from batching. The rest of the model runs in a sample-wise mode, corresponding to batch size B=1𝐵1B=1. The parallelism in the sample-wise computation is dynamically adapted by adjusting the number of parallel iterations based on the size of (T,U)𝑇𝑈(T,U). In [14], an intermediate estimation step is used to select (T,U) positions for pruning, saving memory.Our approach uses sample-wise computation without pruning.

This work presented a sample-wise method to reduce the memory consumption for the computation of transducer loss and gradients. The computation in joint network, output layer and transducer loss is carried out with a batch size of 1. As a result, the memory consumption is almost independent of batch size: we were able to compute the transducer gradients on large batch sizes of up to 1024, and audio lengths of 40 seconds, within less than 6 GB of memory.

The sample-wise processing allows the removal of all padding from individual samples, saving computation. While the sample-wise method comes with an inherent loss of parallelism, some parallelism is recovered by using the parallel for loop in TensorFlow. Here we proposed to adapt the number of parallel iterations dynamically, depending on the input size. This allows one to balance memory consumption and performance, as befits a specific compute/resource environment.

Future work may further improve memory consumption and/or performance by combining sample-wise computation with other techniques such as mixed precision techniques and pruning.
