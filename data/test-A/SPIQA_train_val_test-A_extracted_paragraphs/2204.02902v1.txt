Bayesian networks are arguably the most popular and important model for representing dependencies between variables of multivariate probability distributions [8]. A Bayesian network consists of a directed acyclic graph (DAG) D𝐷D and a set of conditional probability tables, one for each vertex of D𝐷D. The vertices of the network are the variables of the distribution and each conditional probability table specifies the probability distribution of the corresponding vertex given the values of its parents. Thus, the value of each variable depends directly on the values of its parents. The graph D𝐷D is called the structure of the network. Given a set of observed data over some variable set N𝑁N, one needs to learn the structure D𝐷D from this data. This is usually done in two steps. First, for each variable v𝑣v and each possible set of parents a parent score fv​(P)subscript𝑓𝑣𝑃f_{v}(P) is computed. Roughly speaking, the score represents how useful it is to choose this particular set of parents in order to accurately represent the probability distribution that created the observed data. Second, using the computed parent scores, we aim to compute the DAG that maximizes the sum of the parent scores of its vertices. This problem, called Bayesian Network Structure Learning (BNSL), is NP-hard [4].

The current-best worst-case running time bound for exact algorithms for BNSL is 2n⋅poly​(|I|)⋅superscript2𝑛poly𝐼2^{n}\cdot\text{{poly}}(|I|) where n𝑛n is the number of variables and |I|𝐼|I| is the size of the instance. Clearly, this running time is impractical for larger variable sets. Moreover, even very restricted special cases of BNSL remain NP-hard, for example the case where both every possible parent set has constant size and every variable has only a constant number of parent sets [22]. Finally, restricting the topology of the DAG to sparse classes such as graphs of bounded treewidth or bounded degree leads to NP-hard learning problems as well [18, 19, 15].

Due to this notorious hardness of BNSL, it is mostly solved using heuristics. One of the mostsuccessful heuristic approaches relies on local search [24]. More precisely, in this approach one starts with anarcless DAG D𝐷D and adds, removes, or reverses an arc while this results in an increasednetwork score. More recent local search approaches do not use DAGs as solution representations but rather orderings of the variables [3, 20, 23]. This approach is motivated by the fact that given an ordering τ𝜏\tau of the variables, one may greedily find the optimal DAG D𝐷D among all DAGs for which τ𝜏\tau is a topological ordering [22]. Lee and van Beek [20] study a local search approach that is based on an operation, where one interchanges the positions of two consecutive vertices on the ordering as long as this results in an improvement of the network score. We refer to this operation as an inversion. Alonso-Barba et al. [3] an insert operation, where one removes a vertex from the ordering and inserts this vertex at a new position. Scanagatta et al. [23] considered inversions, insertions, and a further generalized operation where one removes a larger block of consecutive vertices from the ordering and inserts the block of consecutive vertices at a new position.

We study different versions of local search on variable orderings. In contrast to previous work that defined the local neighborhood of an ordering as all orderings that can be reached via one operation, we consider parameterized local search [13, 21, 14, 12, 22]. Here, one sets a parameter r𝑟r and aims to find a better network that can be reached via at most r𝑟r modifications of the ordering. The hope is that, by considering such a larger neighborhood, one may avoid being stuck in a bad local optimum. We consider three different kinds of operations in this work. We first study insertions, where one may move one variable to an arbitrary position in the ordering and swaps where two arbitrary variables may exchange their positions. Recall that local search strategies based on one insertion have been studied previously [3, 23]. We observe that the local search problem can be solved in polynomial time for both variants if r𝑟r is constant. The degree of the polynomial, however, depends on r𝑟r. We show that, under widely accepted complexity-theoretic assumptions, this dependence cannot be avoided. Afterwards, we study inversions, which are swaps of adjacent vertices. Our main result is an algorithm with running time 2𝒪​(r​log⁡r)⋅poly​(|I|)⋅superscript2𝒪𝑟𝑟poly𝐼2^{\mathcal{O}(\sqrt{r}\log r)}\cdot\text{{poly}}(|I|) for deciding for a given variable ordering, whether there is a better ordering that can be reached via at most r𝑟r inversions. The distance that measures the minimum number of inversions needed to transform one ordering in another is also known as the Kendall tau distance as it is an adaption of Kedall tau rank correlation [17]. We then introduce a new distance that we call inversion-window distance. Intuitively, given an ordering τ𝜏\tau we find an ordering τ′superscript𝜏′\tau^{\prime} that has inversion-window distance at most r𝑟r by partitioning τ𝜏\tau into multiple windows of consecutive vertices and performing up to r𝑟r inversions inside each window. This new distance extends the number of inversions in the following sense: Given a search radius r𝑟r, the search space for orderings with inversion-window distance r𝑟r is potentially larger than the number of orderings one can obtain with at mots r𝑟r inversions. We provide an algorithm with running time 2𝒪​(r​log⁡r)⋅poly​(|I|)⋅superscript2𝒪𝑟𝑟poly𝐼2^{\mathcal{O}(\sqrt{r}\log r)}\cdot\text{{poly}}(|I|) that decides for a given ordering, whether there is a better ordering that has inversion-window distance at most r𝑟r. Our algorithms work not only for the standard BNSL problem but also for some structuralconstraints that we may wish to impose on the final DAG. To formulate the algorithms compactly,we introduce a generalization of BNSL, where each possible parent setis associated with a score and a weight and the aim is to find the highest scoring networkthat does not exceed a specified weight bound. We show that this captures natural types ofstructural constraints. As a side result, we show that a previous polynomial-timealgorithm for acyclic directed superstructures [22] can be generalized to this new more generalproblem.

Finally, we show that for using an ordering-based local search approach in the presence of structural constraints it is essentially necessary, that the corresponding variant of BNSL is polynomial-time solvable on instances where the directed super structure is acyclic. This implies for several important structural constraints that ordering-based local search is unlikely to be useful.

We initiated the study on parameterized ordering-based local search for the task of learning a Bayesian network structure. We studied four distances d𝑑d and classified for which of these distances d𝑑d-Local W-BNSL is FPT and for which distances it is W[1]-hard.

There are several ways of extending our results that seem interesting topics for futureresearch. First, besides the experimental motivation mentioned in Section 4, this work is purely theoretical. Recall that the running time bottleneck in these preliminary experiments was not the combinatorial explosion in the search radius r𝑟r but rather the slow implementation of the insert operation. Thus, one important topic for future work is to investigate how well the theoretical algorithms proposed in this work perform in practice when combined with algorithm engineering tricks like data reduction rules, upper and lower bounds for the solution size, and restarts.

Besides experimental evaluations of the algorithms proposed in this work, it is also interesting to further study theoretical running time improvements. Recall that our subexponential-time algorithms for the inversions distance and the inversion-window distance are closely related to an algorithm for FAST [1]. Feige [11] and Karpinski and Schudy [16] improved this algorithm for FAST by proposing algorithms with running time 2𝒪​(k)⋅poly​(n)⋅superscript2𝒪𝑘poly𝑛2^{\mathcal{O}(\sqrt{k})}\cdot\text{{poly}}(n) instead of 2𝒪​(k​log⁡(k))⋅poly​(n)⋅superscript2𝒪𝑘𝑘poly𝑛2^{\mathcal{O}(\sqrt{k}\log(k))}\cdot\text{{poly}}(n). Furthermore, the results in this work may be extended by considering further distances. One example for another distance could be a q𝑞q-swap distance, where a q𝑞q-swap for some fixed integer q𝑞q is a swap of two vertices on the ordering which positions differ by at most q𝑞q. In a nutshell, we discussed variants of BNSL that are NP-hard even on instances with an acyclic superstructure—in other words—instances, where the topological ordering of the solution is known.
