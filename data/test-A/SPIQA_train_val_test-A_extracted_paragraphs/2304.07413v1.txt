Robustness to adaptive inputs or adversarial attacks has recently emerged as an important desirable characteristic for algorithm design.An adversarial input can be created using knowledge of the model to induce incorrect outputs on widely used models, such as neural networksВ [BCM+13, SZS+14, GSS15, CW17a, MMS+18].Adversarial attacks against machine learning algorithms in practice have also been documented in applications such as network monitoringВ [CBK09], strategic classificationВ [HMPW16], and autonomous navigationВ [PMG16, LCLS17, PMG+17].The need for sound theoretical understanding of adversarial robustness is also salient in situations where successive inputs to an algorithm can be possibly correlated; even if the input is not adversarially generated, a user may need to repeatedly interact with a mechanism in a way such that future updates may depend on the outcomes of previous interactionsВ [MNS11, GHS+12, BMSC17, NY19, AMYZ19].Motivated by both practical needs and a lack of theoretical understanding, there has been a recent flurry of theoretical studies of adversarial robustness.The streaming model of computation has especially received significant attentionВ [BJWY21, HKM+20, WZ21, KMNS21, BHM+21, CGS22, ABJ+22, CGS22, BEO22, ACGS22, ACSS23, DSWZ23, WZZ23].More recently, there have also been a few initial results for dynamic algorithms on adaptive inputs for graph algorithmsВ [Waj20, BKM+22, BvdBG+22].These works explored the capabilities and limits of algorithms for adversaries that were freely able to choose the input based on previous outputs by the algorithm.

However, in many realistic settings, adversarial input is limited in its abilities.For example, adversarial attacks in machine learning are often permitted to only alter the вҖңtrueвҖқ input by a small amount bounded in norm.For the L0subscriptрқҗҝ0L_{0} norm, this restriction means that the adversary can only add a sparse noise to the true input.More generally, it seems reasonable to assume that adversarial input is generated from a source that has bounded computation time or bounded interactions with an honest algorithm.

In this paper, we study algorithms robust to adaptive/adversarial input generated from sources with bounded capabilities.We first study dynamic algorithms for adaptive inputs from a source that is restricted in sparsity.Namely, we consider robust linear algebraic algorithms when the updates to the label can be adversarial but are restricted in sparsity.We then study robust algorithms in the standard centralized setting, where an adversary queries an algorithm in an adaptive manner, but the number of interactions between the adversary and the algorithm is bounded.We first show that combining novel subroutines for each of these problems in conjunction with a simple but elegant idea of using differential privacy to hide the internal randomness of various subroutines previously used byВ [HKM+20, BKM+22, ACSS23] suffices to achieve robust algorithms across these different settings.

We then consider robust algorithms in the standard centralized setting, where an adversary queries an algorithm in an adaptive manner.In many key algorithmic applications, randomization is necessary to achieve fast query time and efficient storage.This necessitates the need for robust versions of these algorithm which can efficiently employ the power of randomness while also being accurate across multiple possibly correlated inputs.Our main parameters of interest are query time and the space used by a robust algorithm compared to their naГҜve, non robust, counterparts.

Finally, we apply techniques from differential privacyВ [HKM+20, KMNS21, BKM+22, ACSS23] to ensure adversarial robustness, by aggregating independent copies of the algorithm via a private median mechanism.Intuitively, the private mechanism hides the internal randomness of the algorithm and therefore prevents the adversary from otherwise choosing a вҖңbadвҖқ input based on knowledge of internal parameters.

Our framework maintains рқ’Ә~вҖӢ(Q)~рқ’Әрқ‘„\widetilde{\mathcal{O}}\left(\sqrt{Q}\right) instances of the non-adaptive data structure and crucially uses differential privacy (DP) to protect the internal randomness of the data structures.In addition to our previous results for dynamic regression and kрқ‘ҳk-cut queries on adaptive input, the technique of using DP to hide randomness has recently been used in the streaming modelВ [HKM+20, KMNS21, ACSS23] and the dynamic modelВ [BKM+22].These works elegantly use the advanced composition property of DP to bound the number of simultaneous algorithms that must be used in terms of the number of times the output changes вҖңsignificantlyвҖқ over the course of the stream.In the streaming model, the robust algorithms proceed by instantiating many вҖңhiddenвҖқ copies of a standard randomized algorithm.As the stream arrives, the algorithms are updated and an answer, aggregated using DP, is reported.Crucially, many of these results exploit the fact that the output answer is monotonic in the stream so that there is a known upper bound on the final output.Thus, the reported answers can only increase by a multiplicative factor at most a logarithmic number of times, which is used to bound the initial number of algorithms which are initialized.In the centralized setting, this can be imagined as setting the parameter Qрқ‘„Q.The main parameter of interest in the streaming literature is the space used by the streaming algorithms, whereas we are concerned with both space usage and query times.Furthermore, stream elements are only accessed one at a time and cannot be processed together unless memory is used.In our case, the dataset is given to us upfront and we can pre-process it to construct a data structure towards solving a centralized problem.

The work [BKM+22] shares many of these ideas: the authors are concerned with dynamic graph algorithms where an adversary can update the graph in an adaptive fashion.Similar tools such as multiple randomized initialization and aggregated responses using DP are utilized.The main difference is their parameters of interest: the goal of [BKM+22] is to have a fast amortized update time across many queries.This necessitates the need to вҖңthrow awayвҖқ existing algorithms and start with fresh randomness at intermittent points.In contrast, we study a centralized setting where the underlying dataset is not updated but we wish to answer Qрқ‘„Q adaptive queries on the dataset.

By the same reasoning, advanced composition can be used to show the sufficiency of maintaining рқ’Ә~вҖӢ(Q)~рқ’Әрқ‘„\widetilde{\mathcal{O}}\left(\sqrt{Q}\right) data structures to answer Qрқ‘„Q adaptive queries in the centralized setting, which gives a rich set of applications.Moreover, to improve the query time of the framework, the privacy amplification of sampling can be further invoked to show that it suffices to output the private median of a small subset, i.e., a subset of size рқ’ӘвҖӢ(logвҒЎQ)рқ’Әрқ‘„\mathcal{O}\left(\log Q\right), of these рқ’Ә~вҖӢ(Q)~рқ’Әрқ‘„\widetilde{\mathcal{O}}\left(\sqrt{Q}\right) data structures.Thus the framework only incurs a logarithmic overhead in query time and an рқ’Ә~вҖӢ(Q)~рқ’Әрқ‘„\widetilde{\mathcal{O}}\left(\sqrt{Q}\right) overhead in space.Surprisingly, the simple framework gives diverse applications for adaptive algorithms on a number of important problems, including estimating matrix-vector norms, adaptive linear regression, adaptive range query search, adaptive nearest neighbor search, and adaptive kernel density estimation, to name a few.For completeness, these applications are discussed in depth in the appendix in SectionВ A.

We emphasize that for several applications of our framework such as adaptive distance estimation or adaptive kernel density estimation, we additionally use specific sophisticated techniques for these problems to further improve our algorithmic guarantees.As a simple example, for adaptive kernel density estimation, we provide a data structure robust to an arbitrary number of adaptive queries, which cannot be handled by the techniques of [BKM+22].

To achieve better pre-processing time for adaptive distance estimation, our main technique is to sample groups of rows from a Hadamard transform and argue that an interaction with a separate group should be considered in separate privacy budgets, effectively arguing that outputting nрқ‘ӣn approximate distances to a single adaptive query only uses one unit of privacy budget.By contrast, our black-box framework charges one unit of privacy budget per approximate distance, so that outputting nрқ‘ӣn approximate distances would use nрқ‘ӣn units of privacy budget.

TheoremВ 1.4 is based on showing that with constant probability, our data structure is accurate on all possible queries in в„қdsuperscriptв„қрқ‘‘\mathbb{R}^{d}.In particular, we first show that our data structure is accurate on a sufficiently fine net of points through a standard union bound argument, which incurs the dрқ‘‘d overhead compared to the space required to handle a single query.We then show that if the algorithm and the kernel function are both Lipschitz, which is true for sampling-based algorithms and a number of standard kernel functions, then accuracy on the net implies accuracy on all possible points in в„қdsuperscriptв„қрқ‘‘\mathbb{R}^{d}.
