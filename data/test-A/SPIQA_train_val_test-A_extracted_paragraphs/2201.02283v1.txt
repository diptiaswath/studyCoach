For binary (0/1) data, the pGMM kernel becomes the resemblance (Jaccard) similarity and GCWS is essentially equivalent to the celebrated minwise hashing algorithmВ (Broder etВ al., 1997, 1998; Li and Church, 2005; Li and KГ¶nig, 2010), with numerous practical applicationsВ (Fetterly etВ al., 2003; Jindal and Liu, 2008; Buehrer and Chellapilla, 2008; Urvoy etВ al., 2008; Dourisboure etВ al., 2009; Forman etВ al., 2009; Pandey etВ al., 2009; Cherkasova etВ al., 2009; Chierichetti etВ al., 2009; Gollapudi and Sharma, 2009; Najork etВ al., 2009; Bendersky and Croft, 2009; Li etВ al., 2011; Shrivastava and Li, 2012; Schubert etВ al., 2014; Fu etВ al., 2015; Pewny etВ al., 2015; Manzoor etВ al., 2016; Raff and Nicholas, 2017; Tymoshenko and Moschitti, 2018; Zhu etВ al., 2019; Lei etВ al., 2020; Thomas and Kovashka, 2020). Note that minwise hashing can also be used to estimate 3-way and multi-way resemblancesВ (Li etВ al., 2010), not limited to only pairwise similarity.

For general non-binary and non-negative data, the development of consistent weighted sampling algorithm in its current form was due toВ Manasse etВ al. (2010); Ioffe (2010), as well as the earlier versions such asВ Gollapudi and Panigrahy (2006). Li (2015) made the observation about the вҖң0-bitвҖқ CWS, i.e.,В (9) and applied it to approximate large-scale kernel machines using linear algorithms. Li (2017) generalized CWS to general data types which can have negative entries and demonstrated the considerable advantage of CWS over random Fourier featuresВ (Rahimi and Recht, 2007; Li and Li, 2021a). From the computational perspective, CWS is efficient in sparse data. For dense data, algorithms based on rejection samplingВ (Kleinberg and Tardos, 1999; Charikar, 2002; Shrivastava, 2016; Li and Li, 2021b) can be much more efficient than CWS. For relatively high-dimensional datasets, the method of вҖңbin-wise CWSвҖқ (BCWS)В (Li etВ al., 2019) would always be recommended. Simply speaking, the basic idea of BCWS is to divide the data matrix into bins and then apply CWS (for sparse data) or rejection sampling (for dense data) in each bin. Finally, we should add that the mathematical explanation of the вҖң0-bitвҖқ approximationВ (9), empirically observed byВ Li (2015), remains an open problem. The recent work byВ Li etВ al. (2021) developed a related algorithm based on extremal processes and mathematically proved the effectiveness of the вҖң0-bitвҖқ approximation for that algorithm, which is closely related to but not CWS.

In this paper, we develop GCWS (generalized CWS) for hashing the pGMM kernel and then apply GCWS for efficiently training neural networks. We name our procedure GCWSNet and show that GCWSNet can often achieve more accurate results compared with the standard neural networks. We also have another beneficial observation, that the convergent speed of GCWSNet can be substantially faster than the standard neural networks. In fact, we notice that GCWS typically achieves a reasonable accuracy in (less than) one epoch. This characteristic of GCWSNet might be a huge advantage. For example, in many applications such as data streams, one of the 10 challenges in data miningВ (Yang and Wu, 2006), each sample vector is only seen once and hence only one epoch is allowed in the training process. In commercial search engines, when training click-through rate (CTR) deep learning models, typically only one epoch is usedВ (Fan etВ al., 2019; Zhao etВ al., 2019; Fei etВ al., 2021; Xu etВ al., 2021) because the training data size is on the petabyte scale and new observations (i.e., user click information) keep arriving at a very fast rate. Additionally, there is another side benefit of GCWSNet, because the computations at the first layer become additions instead of multiplications.

GCWS generates high-dimensional binary sparse inputs. With kрқ‘ҳk hashes and bрқ‘Ҹb bits for encoding each hashed value, we obtain a vector of size 2bГ—ksuperscript2рқ‘Ҹрқ‘ҳ2^{b}\times k with exactly kрқ‘ҳk 1вҖҷs, for each original data vector. While the (online) training cost is mainly determined by kрқ‘ҳk the number of nonzero entries, the model size is still proportional to 2bГ—ksuperscript2рқ‘Ҹрқ‘ҳ2^{b}\times k. This might be an issue for the GPU memory when both kрқ‘ҳk and bрқ‘Ҹb are large. In this scenario, the well-known method of вҖңcount-sketchвҖқ might be helpful for reducing the model size.

Count-sketchВ (Charikar etВ al., 2004) was originally developed for recovering sparse signals (e.g., вҖңelephantsвҖқ or вҖңheavy hittersвҖқ in network/database terminology). Weinberger etВ al. Even though GCWSNet is obviously not a tree algorithm, there might be some interesting hidden connections. Basically, in GCWS, data entries with (relatively) higher values would be more likely to be picked (i.e., whose locations are chosen as iвҲ—superscriptрқ‘–i^{*}вҖҷs) in a probabilistic manner. Recall that in treesВ (Brieman etВ al., 1983; Hastie etВ al., 2001), only the relative orders of the data values matter, i.e., trees are invariant to any monotone transformation. GCWS is somewhat in-between. It is affected by monotone transformations but not вҖңtoo muchвҖқ.

Naively applying power transformation on the original data might encounter practical problems. For example, when data entries contain large values such as вҖң1533вҖқ or вҖң396вҖқ, applying a powerful transformation such as p=80рқ‘қ80p=80 might lead to overflow problems and other issues such as loss of accuracy during computations. The method of random Fourier features is very popular in academic research. In comparison, research activities on consistent weighted sampling and variants are sparse. We hope this study might generate more interest in GCWS and motivate researchers as well as practitioners to try this interesting method.

In this paper, we propose the pGMM kernel with a single tuning parameter pрқ‘қp and use GCWS (generalized consistent weighted sampling) to generate hash values, producing sparse binary data vectors of size 2bГ—ksuperscript2рқ‘Ҹрқ‘ҳ2^{b}\times k, where kрқ‘ҳk is the number of hash samples and bрқ‘Ҹb is the number of bits to store each hash value. These binary vectors are fed to neural networks for subsequent tasks such as regression or classification. Many interesting findings can be summarized which might benefit future research or practice.

Experiments show that GCWS converges fast and typically reaches a reasonable accuracy at (less than) one epoch. This characteristic of GCWS could be highly beneficial in practice because many important applications such as data streams or CTR predictions in commercial search engines train only one epoch.

GCWS with a single tuning parameter pрқ‘қp provides a beneficial and robust power transformation on the original data. By adding this tuning parameter, the performance can often be improved, in some cases considerably so. Inside GCWS, this tuning parameter acts on the (nonzero) data entry as pвҖӢlogвҒЎuрқ‘қрқ‘ўp\log u (which is typically robust). The nature of GCWS is that the coordinates of entries with larger values have a higher chance to be picked as output (in a probabilistic manner). In other words, both the absolute values and relative orders matter, and the power transformation does impact the performance unlike trees which are invariant to monotone transformations. Experiments show that GCWSNet with a tuning parameter pрқ‘қp produces more accurate results compared with two obvious strategies: 1) feeding power-transformed data (upsuperscriptрқ‘ўрқ‘қu^{p}) directly to neural nets; 2) feeding log-power-transformed data (pвҖӢlogвҒЎuрқ‘қрқ‘ўp\log u, with zeros handled separately) directly to neural nets.

GCWSNet can be combined with count-sketch to reduce the model size of GCWSNet, which is proportional to 2bГ—ksuperscript2рқ‘Ҹрқ‘ҳ2^{b}\times k and can be pretty large if bрқ‘Ҹb is large such as 16 or 24.
