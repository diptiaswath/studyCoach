There are several ways to reduce the communication in SGD. For instance by simply increasing the amount of computation before communication, i.e. by using large mini-batches (see e.g.В [43, 12]), or by designing communication-efficient schemesВ [45]. These approaches are a bit orthogonal to the methods we consider in this paper, which focus on quantization or sparsification of the gradient.

Several papers consider approaches that limit the number of bits to represent floating point numbersВ [13, 24, 31].Recent work proposes adaptive tuning of the compression ratioВ [7]. Unbiased quantization operators not only limit the number of bits, but quantize the stochastic gradients in such a way that they are still unbiased estimators of the gradientВ [3, 41]. The ZipML framework also applies this technique to the dataВ [44].Sparsification methods reduce the number of non-zero entries in the stochastic gradientВ [3, 40].

A very aggressive sparsification method is to keep only very few coordinates of the stochastic gradient by considering only the coordinates with the largest magnitudesВ [9, 1]. In contrast to the unbiased schemes it is clear that such methods can only work by using some kind of error accumulation or feedback procedure, similar to the one the we have already discussedВ [33, 37], as otherwise certain coordinates could simply never be updated. However, in certain applications no feedback mechanism is neededВ [38]. Also more elaborate sparsification schemes have been introducedВ [21].

Asynchronous updates provide an alternative solution to disguise the communication overhead to a certain amount [19]. However, those methods usually rely on a sparsity assumption on the updatesВ [25, 31], which is not realistic e.g. in deep learning. We like to advocate that combining gradient sparsification with those asynchronous schemes seems to be a promising approach, as it combines the best of both worlds. Other scenarios that could profit from sparsification are heterogeneous systems or specialized hardware, e.g. acceleratorsВ [11, 44].

Convergence proofs for SGDВ [29] typically rely on averaging the iteratesВ [30, 27, 23], though convergence of the last iterate can also be provenВ [34]. For our convergence proof we rely on averaging techniques that give more weight to more recent iteratesВ [17, 34, 28],as well as the perturbed iterate framework from Mania et al.В [22] and techniques fromВ [18, 36].

We provide the first concise convergence analysis of sparsified SGDВ [37, 33, 9, 1]. This extremely communication-efficient variant of SGD enforces sparsity of the applied updates by only updating a constant number of coordinates in every iteration. This way, the method overcomes the communication bottleneck of SGD, while still enjoying the same convergence rate in terms of stochastic gradient computations.

Our experiments verify the drastic reduction in communication cost by demonstrating that Mem-SGD requires one to two orders of magnitude less bits to be communicated than QSGDВ [3] while converging to the same accuracy.The experiments show an advantage for the top-kрқ‘ҳk sparsification over random sparsification in the serial setting, but not in the multi-core shared memory implementation. There, both schemes are on par, and show better scaling than a simple shared memory implementation that just writes the unquantized updates in a lock-free asynchronous fashion (like Hogwild!В [25]).

The theoretical insights to Mem-SGD that were developed here should facilitate the analysis of the same scheme in the parallel (as developped inВ [8]) and the distributed setting.It has already been shown in practice that gradient sparsification can be efficiently applied to bandwidth memory limited systems such as multi-GPU trainingfor neural networksВ [9, 1, 21, 37, 33]. By delivering sparsity no matter if the original gradients were sparse or not, our scheme is not only communication efficient, but becomes more eligible for asynchronous implementations as well. While those were so far limited by strict sparsity assumptions (as e.g. inВ [25]), our approach might make such methods much more widely applicable.
