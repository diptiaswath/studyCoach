PCA is one of the most fundamental algorithmic tools for analyzing large data sets. Given a data matrix 𝐀∈ℝn×d𝐀superscriptℝ𝑛𝑑\mathbf{A}\in\mathbb{R}^{n\times d} and a parameter k𝑘k the classic principal component analysis (PCA) problem asks to compute the top k𝑘k eigenvectors of 𝐀⊤​𝐀superscript𝐀top𝐀\mathbf{A}^{\top}\mathbf{A}. This is a core computational task in machine learning and often used for feature selection [17, 32, 29], data visualization [24, 18], and model compression [34].

However, as k𝑘k becomes large, the running time of PCA can degrade. Even just writing down the output takes Ω​(k​d)Ω𝑘𝑑\Omega(kd) time and the performance of many methods degrade with k𝑘k.This high-computational cost for exploring large-cardinality top-eigenspaces has motivated researchers to consider prominent tasks solved by PCA, for example principal component projection (PCP) which asks to project a vector onto the top-k𝑘k eigenspace, and principal component regression (PCR) which asks to solve regression restricted to this top-k𝑘k eigenspace (see Section 1.2).

Recent work [10, 5] showed that the dependence on k𝑘k in solving PCP and PCR can be overcome by instead depending on eigengap γ𝛾\gamma, defined as the ratio between the smallest eigenvalue in the space projected onto and the largest eigenvalue of the space projected orthogonal to. These works replace the typical poly​(k)​nnz​(𝐀)poly𝑘nnz𝐀\mathrm{poly}(k)\textup{nnz}(\mathbf{A}) dependence in runtime with a poly​(1/γ)​nnz​(𝐀)poly1𝛾nnz𝐀\mathrm{poly}(1/\gamma)\textup{nnz}(\mathbf{A}) (at the cost of lower order terms), by reducing these problems to solving poly​(1/γ)poly1𝛾\mathrm{poly}(1/\gamma) ridge-regression problems. Unfortunately, for large-scale problems, as data-set sizes grow so too can k𝑘k and 1/γ1𝛾1/\gamma, yielding large super-linear running times for all previously known methods (see Section 1.4). Consequently, this leaves the following fundamental open problem:Can we obtain nearly linear running times for solving PCP and PCR to high precision, i.e. having running time O~​(nnz​(𝐀))~𝑂nnz𝐀\tilde{O}(\textup{nnz}(\mathbf{A})) plus an additive term depending only on the eigenvalue distribution?

The main contribution of the paper is an affirmative answer to this question. We design randomized algorithms that solve PCP and PCR with high probability in nearly linear time. Leveraging rational polynomial approximations we reduce these problems to solving asymmetric linear systems, which we solve by a technique we call asymmetric SVRG. Further, we provide experiments demonstrating the efficacy of this method.

To obtain our results, critically we depart from the previous frameworks in Frostig et al. [10], Allen-Zhu and Li [5] for solving PCP and PCR. These papers use polynomial approximations to the sign function to reduce PCP and PCR to solving ridge regression. Their runtime is limited by the necessary Ω​(1/γ)Ω1𝛾\Omega(1/\gamma) degree for polynomial approximation of the sign function shown by Eremenko and Yuditskii [8]. Consequently, to obtain nearly linear runtime, a new insight is required.

In our paper, we instead consider rational approximations to the sign function and show that these efficiently reduce PCP and PCR to solving a particular class of squared linear systems.The closed form expression for the best rational approximation to sign function was given by Zolotarev [35] and has recently been proposed for matrix sign approximation [20]. The degree of such rational functions is logarithmic in 1/γ1𝛾1/\gamma, leading to much fewer linear systems to solve. In this paper we provided a new linear algebraic primitive, asymmetric SVRG for solving squared ridge regression problems, and showed that it lead to nearly-linear-time algorithms for PCP and PCR. Beyond the direct running time improvements, this work shows that running time improvements can be achieved for fundamental linear-algebraic problems by leveraging stronger subroutines than standard ridge regression. The improvements we obtain for PCP, demonstrated theoretically and empirically, we hope are just the first instance of a more general approach for improving the running time for solving large-scale machine learning problems.
