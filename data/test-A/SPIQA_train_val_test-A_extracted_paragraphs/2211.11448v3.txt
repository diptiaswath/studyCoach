StyleGAN [32, 31, 30] achieves numerous successes in image generation. Its semantically disentangled latent space enables attribute-based image editing where image content is modified based on the semantic attributes. GAN inversion [64] projects an input image into the latent space, which benefits a series of real image editing methods [4, 51, 37, 68, 75]. The crucial part of GAN inversion is to find the inversion space to avoid distortion while enabling editability. Prevalent inversion spaces include the latent space 𝒲+superscript𝒲\mathcal{W^{+}} [1] and the feature space ℱℱ\mathcal{F} [29]. 𝒲+superscript𝒲\mathcal{W^{+}} is shown to balance distortion and editability [58, 74]. It attracts many editing methods [1, 2, 21, 55, 5, 26] to map real images into this latent space. On the other hand, ℱℱ\mathcal{F} contains spatial image representation and receives extensive studies from the image embedding [29, 50, 65, 61] or StyleGAN’s parameters [6, 14] perspectives.

The latent space 𝒲+superscript𝒲\mathcal{W^{+}} and feature space ℱℱ\mathcal{F} receive wide investigations. In contrast, Karras et al. [32] put into exploring 𝒲𝒲\mathcal{W} and the results are unsatisfying. This may be because that manipulation in 𝒲𝒲\mathcal{W} will easily bring content distortions during reconstruction [58], even though 𝒲𝒲\mathcal{W} is effective for editability. Nevertheless, we observe that 𝒲+superscript𝒲\mathcal{W^{+}} and ℱℱ\mathcal{F} are indeed developed from 𝒲𝒲\mathcal{W}, which is the foundation latent space in StyleGAN. In order to improve image editability while maintaining reconstruction fidelity (i.e., 𝒲+superscript𝒲\mathcal{W^{+}} and ℱℱ\mathcal{F}), exploring 𝒲𝒲\mathcal{W} is necessary. Our motivation is similar to the following quotation:

“You can’t build a great building on a weak foundation. You must have a solid foundation if you’re going to have a strong superstructure.”

—Gordon B. Hinckley

In this paper, we propose a two-step design to improve the representation ability of the latent code in 𝒲+superscript𝒲\mathcal{W^{+}} and ℱℱ\mathcal{F}. First, we obtain the proper latent code in 𝒲𝒲\mathcal{W}. Then, we use the latent code in 𝒲𝒲\mathcal{W} to guide the latent code in 𝒲+superscript𝒲\mathcal{W^{+}} and ℱℱ\mathcal{F}. In the first step, we propose a contrastive learning paradigm to align the 𝒲𝒲\mathcal{W} and image space. This paradigm is derived from CLIP [53] where we switch the text branch with 𝒲𝒲\mathcal{W}. Specifically, we construct the paired data that consists of one image I𝐼I and its latent code w∈𝒲𝑤𝒲w\in\mathcal{W} with pre-trained StyleGAN. During contrastive learning, we train two encoders to obtain two feature representations of I𝐼I and w𝑤w, respectively. These two features are aligned after the training process. During GAN inversion, we fix this contrastive learning module and regard it as a loss function. This loss function is set to make the one real image and its latent code w𝑤w sufficiently close. This design improves existing studies [32] on 𝒲𝒲\mathcal{W} that their loss functions are set on the image space (i.e., similarity measurement between an input image and its reconstructed image) rather than the unified image and latent space. The supervision on the image space only does not enforce well alignment between the input image and its latent code in 𝒲𝒲\mathcal{W}.

After discovering the proper latent code in 𝒲𝒲\mathcal{W}, we leverage a cross-attention encoder to transform w𝑤w into w+∈𝒲+superscript𝑤superscript𝒲w^{+}\in\mathcal{W^{+}} and f∈ℱ𝑓ℱf\in\mathcal{F}. When computing w+superscript𝑤w^{+}, we set w𝑤w as the query and w+superscript𝑤w^{+} as the value and key. Then, we calculate the cross-attention map to reconstruct w+superscript𝑤w^{+}. This cross-attention map enforces the value w+superscript𝑤w^{+} close to the query w𝑤w, which enables the editability of w+superscript𝑤w^{+} to be similar to that of w𝑤w. Besides, w+superscript𝑤w^{+} is effective in preserving the reconstruction ability. When computing f𝑓f, we set the w𝑤w as the value and key, while setting f𝑓f as the query. So w𝑤w will guide f𝑓f for feature refinement. Finally, we use w+superscript𝑤w^{+} and f𝑓f in StyleGAN to generate the reconstruction result.

We named our method CLCAE (i.e., StyleGAN inversion with Contrastive Learning and Cross-Attention Encoder). We show that our CLCAE can achieve state-of-the-art performance in both reconstruction quality and editing capacity on benchmark datasets containing human portraits and cars. GAN inversion [73] is the task to find a latent code in a latent space of pretrained-GAN’s domain for the real image. As mentioned in the GAN inversion survey [64], the inversion methods can be divided into three groups: optimization-based, encoder-based, and hybrid. The optimization-based methods [1, 2, 11, 7, 11, 20, 66, 74] try to directly optimize the latent code or the parameters of GAN [55] to minimize the distance between the reconstruction image. The encoder-based methods [54, 29, 10, 5, 58, 21, 34, 26, 46, 52]learn a mapper to transfer the image to the latent code. The hybrid methods [72, 73] combine these two methods.StyleGAN Inversion.Our work belongs to the StyleGAN inversion framework. Typically, there are three embedding spaces (i.e., 𝒲𝒲\mathcal{W} [31], 𝒲+superscript𝒲\mathcal{W^{+}} [1], and ℱℱ\mathcal{F} [29]) and they are the trade-off design between the distortion and editability. The 𝒲𝒲\mathcal{W} is the foundation latent space of StyleGAN, several works [58, 74] have shown inverting the image into this space produces a high degree of editability but unsatisfied reconstruction quality. Differently, the 𝒲+superscript𝒲\mathcal{W^{+}} is developed from 𝒲𝒲\mathcal{W} to reduce distortions while suffering less editing flexibility. On the other hand, the ℱℱ\mathcal{F} space consists of specific features in SyleGAN, and these features are generated by the latent input code of foundation latent space 𝒲𝒲\mathcal{W} in the StyleGAN training domain. The ℱℱ\mathcal{F} space contains the highest reconstruction ability, but it suffers the worst editability. Different from these designs that directly explore 𝒲+superscript𝒲\mathcal{W^{+}} and ℱℱ\mathcal{F}, we step back to explore 𝒲𝒲\mathcal{W} and use it to guide 𝒲+superscript𝒲\mathcal{W^{+}} and ℱℱ\mathcal{F} to improve fidelity and editability.

Exploring latent space’s semantic directions improves editing flexibility. Typically, there are two groups of methods to find meaningful semantic directions for latent space based editing: supervised and unsupervised methods. The supervised methods [3, 56, 13, 18] need attribute classifiers or labeled data for specific attributes. InterfaceGAN [56] use annotated images to train a binary Support Vector Machine [47] for each label and interprets the normal vectors of the obtained hyperplanes as manipulation direction.The unsupervised methods [60, 22, 57, 68] do not need the labels. GanSpace [22] find directions use Principal Component Analysis (PCA).Moreover, some methods [25, 51, 62, 75] use the CLIP loss [53] to achieve amazing text guiding image manipulation. And some methods use the GAN-based pipeline to edit or inpaint the images [40, 39, 42, 38, 41].In this paper, we follow the [58] and use the InterfaceGAN and GanSpace to find the semantic direction and evaluate the manipulation performance.

Contrastive learning [23, 19, 16, 49, 8, 17] has shown effective in self-supervised learning. When processing multi-modality data (i.e., text and images), CLIP [53] provides a novel paradigm to align text and image features via contrastive learning pretraining. This cross-modality feature alignment motivates generation methods [33, 63, 51, 62, 75] to edit images with text attributes. In this paper, we are inspired by the CLIP and align the foundation latent space 𝒲𝒲\mathcal{W} and the image space with contrastive learning. Then, we set the contrastive learning framework as a loss function to help us find the suitable latent code in 𝒲𝒲\mathcal{W} for the real image during GAN inversion.

In this section, we first illustrate our implementation details. Then we compare our method with existing methods qualitatively and quantitatively. Finally, an ablation study validates the effectiveness of our contributions. More results are provided in the supplementary files. We will release our implementations to the public.

During the contrastive learning process, we follow the CLIP [53] and use the Adam optimizer [35] to train the image and latent code encoders. We synthesize the image-latent code pair dataset with the pre-trained StyleGAN2 in cars and human portrait domains. We set the batch size to 256 for training. During the StyleGAN inversion process, we train and evaluate our method on cars and human portrait datasets. For the human portrait, we use the FFHQ [31] dataset for training and the CelebA-HQ test set [45] for evaluation. For cars, we use the Stanford Cars [36] dataset for training and testing. We set the resolution of the input image as 256×256256256256\times 256. We follow the pSp [54] and use the Ranger optimizer to train our encoder for GAN inversion, the Ranger optimizer is a combination of Rectified Adam [43] with the Lookahead technique [69]. We set the batch size to 32 during training. We use 8 Nvidia Telsa V100 GPUs to train our model.

Our CLCAE improves the representation ability of the latent code in 𝒲+superscript𝒲\mathcal{W^{+}} and feature in ℱℱ\mathcal{F} spaces. we propose a novel GAN inversion method CLCAE that revisits the StyleGAN inversion and editing from the foundation space 𝒲𝒲\mathcal{W} viewpoint. CLCAE adopts a contrastive learning pre-training to align the image space and latent code space first. And we formulate the pre-training process as a loss function ℒalignsubscriptℒalign\mathcal{L}_{\text{align}} to optimize latent code w𝑤w in 𝒲𝒲\mathcal{W} space during inversion. Finally, CLCAE sets the w𝑤w as the foundation to obtain the proper w+superscript𝑤w^{+} and f𝑓f with proposed cross-attention blocks. Experiments on human portrait and car datasets prove that our method can simultaneously produce powerful w𝑤w, w+superscript𝑤w^{+}, and f𝑓f. In the future, we will try to expand this contrastive pre-training process to other domains (e.g., Imagenet dataset [12]) anddo some basic downstream tasks such as classification and segmentation. This attempt could bring a new perspective to contrastive learning.
