Our coreset construction is based on the importance sampling frameworkintroduced by Feldman and Langberg [18]and subsequently improved and generalized by [21, 8].In the framework, one first computes an importance score σxsubscript𝜎𝑥\sigma_{x} for every data point x∈X𝑥𝑋x\in X,and then draws independent samples with probabilitiesproportional to these scores.When no values are missing, the importance scores can be computed easily,even for general metric spaces [50, 21, 8].However, a significant challenge with missing valuesis that distances do not satisfy the triangle inequality,hence importance scores cannot be easily computed.

We overcome this hurdle using a method introducedby Varadarajan and Xiao [49] for projective clustering(where the triangle inequality similarly does not hold).They reduce the importance-score computation to the constructionof a coreset for k𝑘k-Center objective;this method is quite different from earlier approaches,e.g. [18, 50, 21, 8],and yields a coreset for k𝑘k-Means whose size depends linearly on log⁡n𝑛\log nand of course on the size of the k𝑘k-Center coreset.(Mathematically, this arises from the sum of all importance scores.)We make use of this reduction, and thus focus on constructing (efficiently)a small coreset for k𝑘k-Center with missing values.

An immediate difficulty is how to deal with the missing values.We show that it is possible to find a collection of subsets of coordinates ℐℐ\mathcal{I} (so each I∈ℐ𝐼ℐI\in\mathcal{I} is a subset of [d]delimited-[]𝑑[d]),such that if we construct k𝑘k-Center coresets SIsubscript𝑆𝐼S_{I} on the data set “restricted” to each I∈ℐ𝐼ℐI\in\mathcal{I},then the union of these SIsubscript𝑆𝐼S_{I}’s is a k𝑘k-Center coreset for the original data set with missing values.Crucially, we ensure that each “restricted” data set does not contain any missing value,so that it is possible to use a classical coreset construction for k𝑘k-Center.Finally, we show in a technical lemma how to find a collection as necessary of size |ℐ|≤(j​k)O​(min⁡{j,k})ℐsuperscript𝑗𝑘𝑂𝑗𝑘|\mathcal{I}|\leq(jk)^{O(\min\{j,k\})}.

Since a “restricted” data set does not contain any missing values,we can use a classical k𝑘k-Center coreset construction,and a standard construction has size O​(k​ϵ−d)𝑂𝑘superscriptitalic-ϵ𝑑O(k\epsilon^{-d}) [2], which is known to be tight.We bypass this ϵ−dsuperscriptitalic-ϵ𝑑\epsilon^{-d} limitation by observing thatactually O~​(1)~𝑂1\tilde{O}(1)-coreset for k𝑘k-Center suffices,even though the final coreset error is ϵitalic-ϵ\epsilon.We observe that an O~​(1)~𝑂1\tilde{O}(1)-coreset can be constructed using a variant ofGonzalez’s algorithm [24].

Coresets for k𝑘k-Means and k𝑘k-Median clustering have been studied extensively for two decades, and we only list a few notable results.The first strong coresets for Euclidean k𝑘k-Means and k𝑘k-Median were given in [30].In the last decade, most work on coresets for clustering follows the importance sampling framework initiated in [39, 18].In Euclidean space, recent work showed that coresets for k𝑘k-Means and k𝑘k-Median clustering can have size that is independent of the Euclidean dimension [21, 47, 31].Beyond Euclidean space, coresets of size independent of the data-set size were constructed also for many important metric spaces [27, 8, 10].A more comprehensive overview can be found in recent surveys [43, 16].

Recently, attention was given also to non-traditional settings of coresets for clustering,includingcoresets for Gaussian mixture models (GMM) [36, 17];simultaneous coresets for a large family of cost functions that include both k𝑘k-Median and k𝑘k-Center [7];and coresets for clustering under fairness constraints [28].Also considered were settings that capture uncertainty,for example when each point is only known to lie in a line (i.e., clustering lines) [41], and when each point comes from a finite set (i.e., clustering point sets) [33].

Our coreset construction builds upon the sensitivity-sampling method (cf. [18]).However, a central technical challenge is that the standard method to compute the sensitivity scores breaks,because distances between points with missing values do not satisfy the triangle inequality.We overcome this using another known method, of [49], that requires a coreset for k𝑘k-Center.Our main innovation is a near-linear time algorithm that computes anO​(1)𝑂1O(1)-approximate k𝑘k-Center coreset for points with missing values.To this end, we need the following key steps, which constitute our main technical contribution.•We reduce the k𝑘k-Center coreset construction with missing values,to the construction of traditional k𝑘k-Center coresets (i.e., without missing values) on a series of instances.These instances are built by restricting data points with missing valuesto a carefully-chosen collection of subspaces.The guarantee needed from this collection is a certain combinatorial structure, and we indeed prove it exists.•The method of Varadarajan and Xiao executesthe k𝑘k-Center coreset algorithm many times, and overall takes quadratic time.To improve the running time, we design an efficient dynamic algorithmfor the well-known Gonzales’ algorithm(which computes an O​(1)𝑂1O(1)-approximate k𝑘k-Center coreset).The main idea in this dynamic algorithm is to project the data points onto (data-oblivious) random 1D lines, and build on each line a dynamic data structure that supports furthest-neighbor queries (in 1D).Finally, we implemented our algorithm and the experiments indicate that our algorithm is efficient and accurate enough to be potentially applicable in practice.

As an immediate follow-up, one could try to improve our coreset size,e.g., removing the dependence in log⁡n𝑛\log n.Our input can be viewed as axis-parallel affine-subspaces.Hence, another an interesting direction is to obtain coresets for the more general setting where the input consists of general affine-subspaces.
