In many distributed learning applications, a server seeks to compute population information about data that is distributed across multiple clients (users).For example, consider a distributed frequency estimation problem where there are nрқ‘ӣn clients, each holding a local data from a domain of size dрқ‘‘d, and a server that aims to estimate the frequency of the items from the nрқ‘ӣn clients with the minimum communication cost.This task can be done efficiently by letting each client binary encode their data and send the encoding to the server, at a local communication bandwidth cost of logвҒЎ(d)рқ‘‘\log(d) bits.With the binary encoding, the server can faithfully decode each local data and compute the global frequency vector (i.e., the normalized histogram vector).

However, the local data could be sensitive or private, and the clients may wish to keep it hidden from the server.The above binary encoding communication method, unfortunately, allows the server to observe each individual local data, and therefore may not satisfy the usersвҖҷ privacy concerns.Federated Analytics (FA) [Ramage and Mazzocchi, 2020, Zhu etВ al., 2020] addresses this issue by developing new methods that enable the server to learn population information about the clients while preventing the server from prying on any individual local data.In particular, a cryptographic multi-party computation protocol, Secure Summation (рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum}) [Bonawitz etВ al., 2016], has become a widely adopted solution to provide data minimization guarantees for FA [Bonawitz etВ al., 2021].Specifically, рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum} sets up a communication protocol between clients and the server, which injects carefully designed additive noise to each data that cancels out when all of the local data are summed together, but blurs out (information theoretically) each individual local data.Under рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum}, the server is able to faithfully obtain the correct summation of the data from all clients but is unable to read a single local data.The federated frequency estimation (FFE) problem refers to the distributed frequency estimation problem under the constraint of рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum}.Clearly, the binary encoding method is not compatible with рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum}, because when the binary encoding is passed to the server through рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum}, the server only gets the summation of the binary encodings of the usersвҖҷ data, which does not provide sufficient information for computing the global frequency vector.

A naive approach to FFE can be accomplished by employing one-hot encoding: each client encodes its local data into a dрқ‘‘d-dimensional one-hot vector that represents the local frequency vector and sends it to the server through рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum}.Then the server observes the summation of the local frequency vectors using рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum} and scales it by the number of clients to obtain the true frequency vector.However, this one-hot encoding approach costs ОҳвҖӢ(dвҖӢlogвҒЎ(n))Оҳрқ‘‘рқ‘ӣ\Theta(d\log(n)) bits of communication per client. This is because рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum} adds noise from a finite group of size ОҳвҖӢ(logвҒЎn)Оҳрқ‘ӣ\Theta(\log n) to each component of the dрқ‘‘d-dimensional local frequency vector [Bonawitz etВ al., 2016] to avoid overflows.With a linear dependence on domain size dрқ‘‘d, the one-hot encoding approach is inefficient for large domain problems, which is a common setting in practice.In what follows, we focus on the regime where d>nрқ‘‘рқ‘ӣd>n.

Recently, linear compression methods were applied to mitigate the high communication cost issue for FFE with large domains [Chen etВ al., 2021, 2022].The idea is to first linearly compress the local frequency vector into a lower dimensional vector before sending it to the server through рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum}; as linear compression operators commute with the summation operator, the server equivalently observes a linearly compressed global frequency vector though рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum} (after rescaling by the number of clients).The server then applies standard decoding methods to approximately recover the global frequency vector from the linearly compressed one.In particular, Chen etВ al. [2022] show that рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{CountSketch} [Charikar etВ al., 2002] (among other sparse recovery methods) can be used as a linear compressor for the above purpose, which leads to a communication bandwidth cost of рқ’ӘвҖӢ(nвҖӢlogвҒЎ(d)вҖӢlogвҒЎ(n))рқ’Әрқ‘ӣрқ‘‘рқ‘ӣ\mathcal{O}(n\log(d)\log(n)) bits.Therefore when d>nрқ‘‘рқ‘ӣd>n, рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{CountSketch} achieves a saving in local communication bandwidth compared to the one-hot encoding method that requires ОҳвҖӢ(dвҖӢlogвҒЎ(n))Оҳрқ‘‘рқ‘ӣ\Theta(d\log(n)) bits.Moreover, Chen etВ al. [2022] show that for FFE with a single communication round, an О©вҖӢ(nвҖӢlogвҒЎ(d))О©рқ‘ӣрқ‘‘\Omega(n\log(d)) local communication cost is information-theoretically unavoidable for worst-case data distributions, i.e., we cannot do better without making additional assumptions on the global frequency vector.

In this work, we make three notable extensions to рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{CountSketch} for FFE problems.1.We show that the way Chen etВ al. [2022] set up the sketch size (linear in the number of clients nрқ‘ӣn) is often pessimistic (see Corollary 2.4).In fact, in the streaming literature, the estimation error induced by рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{CountSketch} is known to adapt to the tail norm of the global frequency vector [Minton and Price, 2014], which is often sub-linear in nрқ‘ӣn.Motivated by this, we provide an easy-to-use, two-phase approach that allows practitioners to determine the necessary sketch size by automatically adapting to the hardness of the FFE problem instance.2.We consider FFE with multiple communication rounds, which better models practical deployments of FA where aggregating over (hundreds of) millions of clients in a single round is not possible due to device availability and limited server bandwidth.We propose a new multi-round sketch algorithm called рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{HybridSketch} that provably performs better than simple adaptations of рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{CountSketch} in the multi-round setting, leading to further improvements in the communication cost.Surprisingly, we show that рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{HybridSketch} adapts to the tail norm of a heterogeneity vector (see Theorem 3.2).Moreover, the tail of the heterogeneity vector is always no heavier, and could be much lighter, than that of the global frequency vector, explaining the advantage of рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{HybridSketch}. For instance, on the C4 dataset [Bowman etВ al., 2020] with a domain size of d=150,868рқ‘‘150868d=150,868 and 150,000150000150,000 users, we show that our method can reduce the sketch size by 83%percent8383\% relative to simple sketching methods when the number of sketch rows is not very large.3.We extend the Gaussian mechanism for рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{CountSketch} proposed by Pagh and Thorup [2022], Zhao etВ al. [2022] to the multi-round FFE setting to show how our sketching methods can be made differentially private [Dwork etВ al., 2006].We also characterize the trade-offs between accuracy and privacy for our proposed method.

Hybrid Sketch.Both рқҡӮрқҡ‘рқҡҠрқҡӣрқҡҺрқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҡӮрқҡ‘рқҡҠрқҡӣрқҡҺрқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{SharedSketch} and рқҷөрқҡӣрқҡҺрқҡңрқҡ‘рқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷөрқҡӣрқҡҺрқҡңрқҡ‘рқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{FreshSketch} reduce a multi-round FFE problem into single-round FFE problem(s).In contrast, we show a more comprehensive sketching method, called count sketch with hybrid hash design (рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{HybridSketch}), that solves a multi-round FFE problem as a whole.рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{HybridSketch} is presented as Algorithm 2.Specifically, рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{HybridSketch} generates Mрқ‘ҖM sketches that share a set of bucket hashes but use independent sets of sign hashes.Then in the mрқ‘ҡm-th communication round, participating clients and the server communicate by the рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷІрқҡҳрқҡһрқҡ—рқҡқрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{CountSketch} algorithm based on the mрқ‘ҡm-th sketch, sothe server observes the summation of the sketched data through рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum}.After collecting Mрқ‘ҖM summations of the sketched local data, the server first computes averages over different rounds for variance reduction, then computes the median over different repeats (or sketch rows) for success probability amplification.We provide the following problem-dependent bound for рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{HybridSketch}.

We would like to point out that, although our рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘рқҷ·рқҡўрқҡӢрқҡӣрқҡ’рқҡҚрқҡӮрқҡ”рқҡҺрқҡқрқҡҢрқҡ‘\mathtt{HybridSketch} algorithm is developed for multi-round frequency estimation problems, it can be adapted to multi-round vector recovery problems as well. Hence it could have broader applications in other federated learning scenarios.

While рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–рқҡӮрқҡҺрқҡҢрқҡӮрқҡһрқҡ–\mathtt{SecSum} provides security guarantees, it does not provide differential privacy guarantees. In this part, we discuss a simple modifications to the sketching algorithms to make them provably differentially private (DP).

In our case, a dataset corresponds to all participated clients (or their data), and two neighboring datasets should be regarded as two sets of clients (local data) that only differ in a single client (local data).The algorithm refers to all procedures before releasing the final frequency estimate, and all the intermediate computation is considered private and is not released.

Finally, we remark that the trade-off between the estimation error and the DP error in Theorem 4.1 might not be optimal. On the one hand, the DP error in Theorem 4.1 scales with OвҖӢ(1/(nвҖӢM))рқ‘Ӯ1рқ‘ӣрқ‘ҖO(1/(n\sqrt{M})) instead of OвҖӢ(1/(nвҖӢM))рқ‘Ӯ1рқ‘ӣрқ‘ҖO(1/(nM)). This is because we release Mрқ‘ҖM sketched matrices, which might not be necessary for achieving central-DP. We conjecture that our DP mechanism is improvable.On the other hand, one might be able to design multi-round FEE sketching methods that are easier to make differentially private. For instance, one can divide the Mрқ‘ҖM rounds into mвүӨMрқ‘ҡрқ‘Җm\leq M batches and use a shared sketch within each batch but use a hybrid sketch across batches. This modified sketching method can be viewed as applying a hybrid sketch in a multi-round FFE setting of mрқ‘ҡm rounds and nвҖӢM/mрқ‘ӣрқ‘Җрқ‘ҡnM/m participating clients per round. So one can directly apply Theorem 4.1 to it. By tuning mрқ‘ҡm, one might be able to get a better trade-off between the estimation error and the DP error.We leave these issues as open problems for future investigation.

We make several novel extensions to the count sketch method for federated frequency estimation with one or more communication rounds.In the single round setting, we show that count sketch can achieve better communication efficiency when the underlying problem is simpler.We provide a two-phase approach to automatically select a sketch size that adapts to the hardness of the problem.In the multiple rounds setting, we show a new sketching method that provably achieves better accuracy than simple adaptions of count sketch.Finally, we adapt the Gaussian mechanism to make the hybrid sketching method differentially private.

We remark that the improvement of the instance-dependent method relies on the assumptions that the underlying frequency has a lighter tail, which might be unverifiable a priori due to constraints, e.g., limited communication and privacy budget.Finally, this work focuses on an offline setting where the frequency is considered to be fixed. Extending our results to an online setting where the frequency is varying is an interesting future direction.
