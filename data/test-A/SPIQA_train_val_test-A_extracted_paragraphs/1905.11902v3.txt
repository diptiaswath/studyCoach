Clustering is a central problem in unsupervised learning. A clustering problem is typically represented by a set of elements together with a notion of similarity (or dissimilarity) between them. When the elements are points in a metric space, dissimilarity can be measured via a distance function. In more general settings, when the elements to be clustered are members of an abstract set V𝑉V, similarity is defined by an arbitrary symmetric function σ𝜎\sigma defined on pairs of distinct elements in V𝑉V. Correlation Clustering (CC) [4] is a well-known special case where σ𝜎\sigma is a {−1,+1}11\{-1,+1\}-valued function establishing whether any two distinct elements of V𝑉V are similar or not. The objective of CC is to cluster the points in V𝑉V so to maximize the correlation with σ𝜎\sigma. More precisely, CC seeks a clustering minimizing the number of errors, where an error is given by any pair of elements having similarity −11-1 and belonging to the same cluster, or having similarity +11+1 and belonging to different clusters. Importantly, there are no a priori limitations on the number of clusters or their sizes: all partitions of V𝑉V, including the trivial ones, are valid. Given V𝑉V and σ𝜎\sigma, the error achieved by an optimal clustering is known as the Correlation Clustering index, denoted by OPTOPT\mathrm{OPT}.A convenient way of representing σ𝜎\sigma is through a graph G=(V,E)𝐺𝑉𝐸G=(V,E) where {u,v}∈E𝑢𝑣𝐸\{u,v\}\in E iff σ​(u,v)=+1𝜎𝑢𝑣1\sigma(u,v)=+1. Note that OPT=0OPT0\mathrm{OPT}=0 is equivalent to a perfectly clusterable graph (i.e., G𝐺G is the union of disjoint cliques). Since its introduction, CC has attracted a lot of interest in the machine learning community, and has found numerous applications in entity resolution [18], image analysis [20], and social media analysis [27]. Known problems in data integration [15] and biology [5] can be cast into the framework of CC [28].

From a machine learning viewpoint, we are interested in settings when the similarity function σ𝜎\sigma is not available beforehand, and the algorithm must learn σ𝜎\sigma by querying for its value on pairs of objects. This setting is motivated by scenarios in which the similarity information is costly to obtain.For example, in entity resolution, disambiguating between two entities may require invoking the user’s help.Similarly, deciding if two documents are similar may require a complex computation, and possibly the interaction with human experts.In these active learning settings, the learner’s goal is to trade the clustering error against the number of queries.Hence, the fundamental question is: how many queries are needed to achieve a specified clustering error?Or, in other terms, how close can we get to OPTOPT\mathrm{OPT}, under a prescribed query budget Q𝑄Q?

Another line of work attempts to circumvent computational hardness by using the more powerful same-cluster queries (SCQ). A same-cluster query tells whether any two given nodes are clustered together according to an optimal clustering or not. In [3] SCQs are used to design a FPTAS for a variant of CC with bounded number of clusters. In [25] SCQs are used to design algorithms for solving CC optimally by giving bounds on Q𝑄Q which depend on OPTOPT\mathrm{OPT}.Unlike our setting, both works assume all (n2)binomial𝑛2\binom{n}{2} similarities are known in advance.The work [23] considers the case in which there is a latent clustering with OPT=0OPT0\mathrm{OPT}=0.The algorithm can issue SCQs, however the oracle is noisy: each query is answered incorrectly with some probability, and the noise is persistent (repeated queries give the same noisy answer). The above setting is closely related to the stochastic block model (SBM), which is a well-studied model for cluster recovery [1, 21, 24]. However, few works investigate SBMs with pairwise queries [13]. Our setting is strictly harder because our oracle has a budget of OPTOPT\mathrm{OPT} adversarially incorrect answers.

A different model is edge classification. Here the algorithm is given a graph 𝒢𝒢\mathcal{G} with hidden binary labels on the edges. The task is to predict the sign of all edges by querying as few labels as possible [8, 12, 14]. As before, the oracle can have a budget OPTOPT\mathrm{OPT} of incorrect answers, or a latent clustering with OPT=0OPT0\mathrm{OPT}=0 is assumed and the oracle’s answers are affected by persistent noise. Unlike correlation clustering, in edge classification the algorithm is not constrained to predict in agreement with a partition of the nodes. On the other hand, the algorithm cannot query arbitrary pairs of nodes in V𝑉V, but only those that form an edge in 𝒢𝒢\mathcal{G}.
