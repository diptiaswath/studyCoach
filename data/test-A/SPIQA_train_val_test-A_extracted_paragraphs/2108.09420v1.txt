Kernel methods are a powerful tool for solving non-parametric learning problems, such as kernel regression, support vector machines (SVM), principal component analysis (PCA), and many others. A typical burden for kernel methods is that they suffer from scalability, since computing a kernel matrix requires computing a quadratic (in the number of input points) number of entries in the matrix. A direction that has received less attention but still of particular interest is the regime where the dimension d𝑑d of the data points is large. Typically, applying the kernel function to each pair of data points takes O​(d)𝑂𝑑O(d) time. This is especially undesirable in applications for natural language processing [DL20] and computational biology [TPK02], where d𝑑d can be as large as poly⁡(n)poly𝑛\operatorname{poly}(n), with n𝑛n being the number of data points. To compute the kernel matrix, the algorithm does have to read the d×n𝑑𝑛d\times n input matrix. Therefore, algorithms that have a nearly linear dependence on n​d𝑛𝑑nd are of particular interest.

To accelerate the computation of kernel matrices from the naïve O​(n2​d)𝑂superscript𝑛2𝑑O(n^{2}d) time algorithm, a lot of work has focused on finding a good approximation to a kernel matrix efficiently [RR07, AM15, MM17, AKK+20, WZ20]. All of these methods make use of randomized algorithmic primitives such as sampling or sketching. Roughly speaking, the idea is to randomly generate a “sketching matrix" with a small number of rows, multiply the sketching matrix with the input matrix, and show that the resulting matrix approximately preserves the length of vectors in the row or column space of the original matrix.

The polynomial kernel is of interest, since any kernel can be written as a sum of polynomial kernels through a Taylor expansion. If we can efficiently approximate the polynomial kernel, then we will be able to efficiently approximate many types of kernels. In [ANW14], Avron, Nguyen and Woodruff approximate the polynomial kernel with a sketching matrix in time that depends exponentially on the degree p𝑝p of the polynomial kernel. Recent works of [AKK+20, WZ20] have improved the dependence on p𝑝p to polynomial. However, their algorithms mainly focus on optimizing the dependence on n𝑛n and reducing the exponential dependence on p𝑝p to polynomial. If X∈ℝd×n𝑋superscriptℝ𝑑𝑛X\in\mathbb{R}^{d\times n} is the input matrix and is dense, then these algorithms have runtime O~​(p​n​d+ϵ−2​n3​p2)~𝑂𝑝𝑛𝑑superscriptitalic-ϵ2superscript𝑛3superscript𝑝2\widetilde{O}(pnd+\epsilon^{-2}n^{3}p^{2}).111We use O~​(⋅),Ω~​(⋅),Θ~​(⋅)~𝑂⋅~Ω⋅~Θ⋅\widetilde{O}(\cdot),\widetilde{\Omega}(\cdot),\widetilde{\Theta}(\cdot) to suppress poly⁡(log⁡(n​d/ϵ​δ))poly𝑛𝑑italic-ϵ𝛿\operatorname{poly}(\log(nd/\epsilon\delta)) factors. Notice this is unsatisfactory when both d𝑑d and p𝑝p are large.

Thus, a natural question to ask is:Does there exist a sketch for polynomial kernels of degree p𝑝p, such that the runtime is nearly linear in n​d𝑛𝑑nd, and with an improved dependence on p𝑝p?

Notice this is especially desirable for kernels such as the neural tangent kernel (𝖭𝖳𝖪𝖭𝖳𝖪\mathsf{NTK}) [JGH18] and the arc-cosine kernel [CS09], whose Taylor series have a much slower decay rate (1/nc1superscript𝑛𝑐1/n^{c} for some c𝑐c) compared to the Gaussian kernel (which is 1/n!1𝑛1/n!).

Suppose we want to compute the p𝑝p-fold tensoring of a vector x𝑥x, and assume for simplicity that p𝑝p is a power of 222. Our algorithm is inspired by that of [AKK+20], which generates a complete binary tree with 2​p−12𝑝12p-1 nodes, and thus p𝑝p leaves. For the i𝑖i-th leaf node, it picks a sketch Tisuperscript𝑇𝑖T^{i} and applies it to x𝑥x, obtaining Ti​xsuperscript𝑇𝑖𝑥T^{i}x. Each internal node j𝑗j then does the following: it picks a sketch Sjsuperscript𝑆𝑗S^{j}, and applies Sjsuperscript𝑆𝑗S^{j} to the tensor product of its two children. Sjsuperscript𝑆𝑗S^{j} is picked as a map from ℝm2→ℝm→superscriptℝsuperscript𝑚2superscriptℝ𝑚\mathbb{R}^{m^{2}}\rightarrow\mathbb{R}^{m} for each j𝑗j, so at each level of the binary tree, we reduce the number of vectors by half, while remaining in the low-dimensional space ℝmsuperscriptℝ𝑚\mathbb{R}^{m}. One drawback of this algorithm is the usage of an independent sketch for each node of the tree, thus incurring a linear dependence on p𝑝p in the runtime.

Our algorithm instead uses a much smaller amount of randomness. We pick only a single T𝑇T and a single S𝑆S, i.e., T1=T2=⋯=Tpsuperscript𝑇1superscript𝑇2⋯superscript𝑇𝑝T^{1}=T^{2}=\cdots=T^{p} for all leaf nodes, and we have Sj=Ssuperscript𝑆𝑗𝑆S^{j}=S for all internal nodes j𝑗j. The challenge with this approach is of course that we have much less independence in our analysis, and consequently do not obtain the same guarantees for preserving the length of the tensor product of an arbitrary set of vectors as in previous work. We stress that we compute a sketch that preserves the column span of X⊗psuperscript𝑋tensor-productabsent𝑝X^{\otimes p}, and this is weaker than the guarantee we would get had we used full independence, which gives a sketch that is an oblivious subspace embedding, meaning that it can preserve the column span of any matrix in ℝdp×nsuperscriptℝsuperscript𝑑𝑝𝑛\mathbb{R}^{d^{p}\times n}. However, the key point is that we can preserve the tensor product of a vector with itself p𝑝p times, and this will suffice for our applications.

This allows for a much faster way to compute x⊗psuperscript𝑥tensor-productabsent𝑝x^{\otimes p}: “square” a vector by computing the tensor product with itself, then apply a sketch, and repeat this process. By doing so, we reduce the dependence on p𝑝p in the first level of the tree from linear to logarithmic. However, this will incur a p2superscript𝑝2p^{2} factor in the dimension of the sketch, and so we will pay more for p𝑝p in levels other than the first level. Fortunately, levels other than the first level apply sketches to lower dimensional vectors. By carefully balancing the complexity of applying T𝑇T and S𝑆S, we achieve an improved running time, which is useful when the degree p𝑝p is large.
