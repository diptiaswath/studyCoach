GCN is being increasingly used in IR applications, ranging from search enginesВ (MaoetВ al., 2020; Yang, 2020), recommender systemsВ (WangetВ al., 2019a; Chen etВ al., 2019a; HeetВ al., 2020; ChenetВ al., 2020a) to question-answering systemsВ (ZhangetВ al., 2020b; HuetВ al., 2020c).Its main idea is to augment a nodeвҖҷs representation by aggregating the representations of its neighbors.In practice, GCN could face the local structure discrepancy issueВ (ChenetВ al., 2020c) since real-world graphs usually exhibit locally varying structure. That is, nodes can exhibit inconsistent distributions of local structure properties such as homophily and degree.Figure 1 shows an example in a document citation graphВ (Hu etВ al., 2020a), where the local structure centered at nвҖӢoвҖӢdвҖӢeвҖӢ1рқ‘ӣрқ‘ңрқ‘‘рқ‘’1node1 and nвҖӢoвҖӢdвҖӢeвҖӢ2рқ‘ӣрқ‘ңрқ‘‘рқ‘’2node2 has different properties regarding cross-category edges111This work focuses on the discrepancy w.r.t. cross-category connections.. Undoubtedly, applying the same aggregation over nвҖӢoвҖӢdвҖӢeвҖӢ1рқ‘ӣрқ‘ңрқ‘‘рқ‘’1node1 and nвҖӢoвҖӢdвҖӢeвҖӢ2рқ‘ӣрқ‘ңрқ‘‘рқ‘’2node2 will lead to inferior node representations. Therefore, it is essential for GCN to account for the local structure discrepancy issue.

Existing work considers this issue by equipping GCN with an adaptive locality moduleВ (Xu etВ al., 2018; VeliДҚkoviДҮ etВ al., 2018), which learns to adjust the contribution of neighbors.Most of the efforts focus on the attention mechanism, such as neighbor attentionВ (VeliДҚkoviДҮ etВ al., 2018) and hop attentionВ (Liu etВ al., 2020).Ideally, the attention weight could downweigh the neighbors that causes discrepancy, e.g., the neighbors of different categories with the target node.However, graph attention is not easy to be trained well in practice, especially for hard semi-supervised learning setting that has very limited labeled dataВ (KnyazevetВ al., 2019).Moreover, existing methods mainly consider the nodes in the training data, ignoring the local structure discrepancy on the testing nodes, which however are the decisive factor for model generalization. It is thus insufficient to resolve the discrepancy issue by adjusting the architecture of GCN.

In this work, we argue that it is essential to empower the inference stage of a trained GCN with the ability of handling the local structure discrepancy issue.In real-world applications, the graph structure typically evolves along time, resulting in structure discrepancy between the training data and testing data. Moreover, the testing node can be newly coming (e.g., a new user), which may exhibit properties different from the training nodesВ (Tang etВ al., 2020b).However, the one-pass inference procedure of existing GCNsindiscriminately uses the learned model parameters to make prediction for all testing nodes, lacking the capability of handling the structure discrepancy. This work aims to bridge the gap by upgrading the GCN inference to be node-specific according to the extent of structure discrepancy.

To achieve the target, the key lies in analyzing the prediction generation process of GCN on each node and estimating to what extent accounting for nodeвҖҷs neighbors affects its prediction, i.e., the causal effect of the local structure on GCN prediction.According to the evidence that model output can reflect feature discrepancyВ (SaitoetВ al., 2018), we have a key assumption that the GCN output provides evidence on the properties of the local structure centered at a testing node.For instance, if the local structure exhibits properties distinct from the seen ones, the model will be uncertain about its prediction when the neighbors are taken into account.Accordingly, we should downweigh the contribution of neighbors to reduce the impact of the discrepancy on the prediction.Inherently, both a nodeвҖҷs features and neighbors are the causes of the prediction for the node.By distinguishing the two causal effects, we can assess revise the model prediction in a node-specific manner.

To this end, we resort to the language of causal graphВ (Pearl, 2009) to describe the causal relations in GCN prediction. We propose a Causal GCN Inference (CGI) model, which adjusts the prediction of a trained GCN according to the causal effect of the local structure.In particular, CGI first calls for causal intervention that blocks the graph structure and forces the GCN to user a nodeвҖҷs own features to make prediction. CGI then makes choice between the intervened prediction and the original prediction, according to the causal effect of the local structure, prediction confidence, and other factors that characterize the prediction.Intuitively, CGI is expected to choose the intervened prediction (i.e., trusting self) when facing a testing node with local structure discrepancy.To learn a good choice-making strategy, we devise it as a separate classifier, which is learned based on the trained GCN.We demonstrate CGI on APPNPВ (Klicpera etВ al., 2019), one of the state-of-the-art GCN models for semi-supervised node classification. Extensive experiments on seven datasets validate the effectiveness of our approach. The codes are released at: https://github.com/fulifeng/CGI.

The main contributions of this work are summarized as follows:вҖўWe achieve adaptive locality during GCN inference and propose an CGI model that is model-agnostic.вҖўWe formulate the causal graph of GCN working mechanism, and the estimation of causal intervention and causal uncertainty based on the causal graph.вҖўWe conduct experiments on seven node classification datasets to demonstrate the rationality of the proposed methods.

Causal graph is a directed acyclic graph to describe a data generation processВ (Pearl, 2009), where nodes represent variables in the process, and edges represent the causal relations between variables. To facilitate analyzing the inference of GCN, i.e., the generation process of the output, we abstract the inference of GCN as a causal graph (Figure 2(a)), which consists of four variables:вҖўXрқ‘ӢX, which denotes the features of the target node. рқ’ҷрқ’ҷ\bm{x} is an instance of the variable.вҖўNрқ‘ҒN, which denotes the neighbors of the target node, e.g., рқ’©вҖӢ(рқ’ҷ)рқ’©рқ’ҷ\mathcal{N}(\bm{x}). Recall that the conventional GCN inference, i.e., the calculation of рқ’ҡ^^рқ’ҡ\hat{\bm{y}}, is typically a one-pass forward propagation of the GCN model with рқ’ҷрқ’ҷ\bm{x} and рқ’©вҖӢ(рқ’ҷ)рқ’©рқ’ҷ\mathcal{N}(\bm{x}) as inputs.Based on the causal graph, the procedure can be interpreted as Figure 2(b), where every variable obtains an instance (e.g., X=рқ’ҷрқ‘Ӣрқ’ҷX=\bm{x}).Apart from the new understanding of the conventional GCN inference, the causal theory provides analytical tools based on the causal graph, such as causal interventionВ (Pearl, 2009), which enable the in-depth analysis of the factors resulting in the prediction and further reasoning based on the predictionВ (Pearl, 2019).

Note that we take empty set as a representative reference status of N=рқ’©вҖӢ(рқ’ҷ)рқ‘Ғрқ’©рқ’ҷN=\mathcal{N}(\bm{x}) since the widely usage of empty value as reference in causal interventionВ (Pearl, 2009), but can replace it with any subset of рқ’©вҖӢ(рқ’ҷ)рқ’©рқ’ҷ\mathcal{N}(\bm{x}) (see SectionВ 3.3).

The requirement of adaptive locality for the testing nodes pushes us to build up an additional mechanism, i.e., CGI, to enhance the GCN inference stage. We have two main considerations for devising the mechanism: 1)the mechanism has to be learned from the data, instead of handcrafted, to enable its usage on different GCN models and different datasets. 2)the mechanism should effectively capture the connections between the causal effect of N=рқ’©вҖӢ(рқ’ҷ)рқ‘Ғрқ’©рқ’ҷN=\mathcal{N}(\bm{x}) and local structure discrepancy, i.e., learning the patterns for adjusting the original prediction to improve the prediction accuracy.

A straight forward solution is devising the CGI as a LрқҗҝL-way classification model that directly generates the final prediction according to the original prediction рқ’ҡ^^рқ’ҡ\hat{\bm{y}}, the post-intervention prediction рқ’ҡ^ssuperscript^рқ’ҡрқ‘ \hat{\bm{y}}^{s}, and the causal effect рқ’Ҷрқ’Ҷ\bm{e}. Formally,(6)рқ’ҡВҜ=hвҖӢ(рқ’ҡ^,рқ’ҡ^s,рқ’Ҷ|Пү),ВҜрқ’ҡв„Һ^рқ’ҡsuperscript^рқ’ҡрқ‘ conditionalрқ’Ҷрқң”\displaystyle\bar{\bm{y}}=h(\hat{\bm{y}},\hat{\bm{y}}^{s},\bm{e}|\omega),where hвҖӢ(вӢ…)в„ҺвӢ…h(\cdot) denotes a LрқҗҝL-way classifier parameterized by Пүрқң”\omega and рқ’ҡВҜвҲҲв„қLВҜрқ’ҡsuperscriptв„қрқҗҝ\bar{\bm{y}}\in\mathbb{R}^{L} denotes the final prediction. Similar to the training of GCN, we can learn the parameters of hвҖӢ(вӢ…)в„ҺвӢ…h(\cdot) by optimizing classification loss over the labeled nodes, which is formulated as:(7)рққҺ^=minрққҺвҖӢвҲ‘i=1M(lвҖӢ(рқ’ҡВҜi,рқ’ҡi))+ОұвҖӢвҖ–рққҺвҖ–F2,^рққҺsubscriptрққҺsuperscriptsubscriptрқ‘–1рқ‘Җрқ‘ҷsubscriptВҜрқ’ҡрқ‘–subscriptрқ’ҡрқ‘–рқӣјsubscriptsuperscriptnormрққҺ2рқҗ№\displaystyle\hat{\bm{\omega}}=\min_{\bm{\omega}}\sum_{i=1}^{M}\left(l(\bar{\bm{y}}_{i},\bm{y}_{i})\right)+\alpha\|\bm{\omega}\|^{2}_{F},where Оұрқӣј\alpha is a hyperparameter to adjust the strength of regularization.Undoubtedly, this model can be easily developed and applied to any GCN. However, as optimized over the overall classification loss, the model will face the similar issue of attention mechanismВ (KnyazevetВ al., 2019). To bridge this gap, it is essential to learn CGI under the awareness of whether a testing node encounters local structure discrepancy.

Therefore, the inference mechanism should focus on the nodes with inconsistent classifications from рқ’ҡ^^рқ’ҡ\hat{\bm{y}} and рқ’ҡ^ssuperscript^рқ’ҡрқ‘ \hat{\bm{y}}^{s}, i.e., z^вү z^s^рқ‘§superscript^рқ‘§рқ‘ \hat{z}\neq\hat{z}^{s} where z^^рқ‘§\hat{z} is the original classification and z^s=argвҒЎmaxjвҒЎy^(j)s,jвүӨLformulae-sequencesuperscript^рқ‘§рқ‘ subscriptрқ‘—superscriptsubscript^рқ‘Ұрқ‘—рқ‘ рқ‘—рқҗҝ\hat{z}^{s}=\arg\max_{j}~{}\hat{y}_{(j)}^{s},j\leq L is the post-intervention classification. That is, we let the CGI mechanism learn from nodes where accounting for neighbors causes the change of the classification.To this end, we devise the inference mechanism as a choice model, which is expected to make wise choice between z^^рқ‘§\hat{z} and z^ssuperscript^рқ‘§рқ‘ \hat{z}^{s} to eliminate the impact of local structure discrepancy. Formally,(8)zВҜ={z^,p^вүҘt,z^s,p^<t,p^=g(рқ’ҡ^,рқ’ҡ^s,рқ’Ҷ|рқңј),\displaystyle\bar{z}=\left\{\begin{aligned} &\hat{z},\hat{p}\geq t,\\&\hat{z}^{s},\hat{p}<t,\end{aligned}\right.~{}\hat{p}=g(\hat{\bm{y}},\hat{\bm{y}}^{s},\bm{e}|\bm{\eta}),where gвҖӢ(вӢ…)рқ‘”вӢ…g(\cdot) denotes a binary classifier with parameters of рқңјрқңј\bm{\eta}; the output of the classifier pрқ‘қp is used for making choice; and tрқ‘Ўt is the decision threshold, which depends on the classifier selected.

To learn the model parameters рқңјрқңј\bm{\eta}, we calculate the ground truth for making choice according to the correctness of z^^рқ‘§\hat{z} and z^ssuperscript^рқ‘§рқ‘ \hat{z}^{s}. Formally, the choice training data of the binary classifier is:(9)рқ’ҹ={(рқ’ҷ,p)|z^=zвҲӘz^s=z},p=fвҖӢlвҖӢaвҖӢgвҖӢ(z^=z),formulae-sequenceрқ’ҹconditional-setрқ’ҷрқ‘қ^рқ‘§рқ‘§superscript^рқ‘§рқ‘ рқ‘§рқ‘қрқ‘“рқ‘ҷрқ‘Һрқ‘”^рқ‘§рқ‘§\displaystyle\mathcal{D}=\left\{(\bm{x},p)|\hat{z}=z\cup\hat{z}^{s}=z\right\},~{}p=flag(\hat{z}=z),where zрқ‘§z denotes the correct category of node рқ’ҷрқ’ҷ\bm{x}; fвҖӢlвҖӢaвҖӢgвҖӢ(z^=z)=1рқ‘“рқ‘ҷрқ‘Һрқ‘”^рқ‘§рқ‘§1flag(\hat{z}=z)=1 if z^^рқ‘§\hat{z} equals to zрқ‘§z, fвҖӢlвҖӢaвҖӢgвҖӢ(z^=z)=вҲ’1рқ‘“рқ‘ҷрқ‘Һрқ‘”^рқ‘§рқ‘§1flag(\hat{z}=z)=-1 otherwise. The training of the choice model is thus formulated as:(10)рқңј^=minрқңјвҖӢвҲ‘(рқ’ҷ,p)вҲҲрқ’ҹlвҖӢ(p^,p)+ОІвҖӢвҖ–рқңјвҖ–F2,^рқңјsubscriptрқңјsubscriptрқ’ҷрқ‘қрқ’ҹрқ‘ҷ^рқ‘қрқ‘қрқӣҪsubscriptsuperscriptnormрқңј2рқҗ№\displaystyle\hat{\bm{\eta}}=\min_{\bm{\eta}}\sum_{(\bm{x},p)\in\mathcal{D}}l(\hat{p},p)+\beta\|\bm{\eta}\|^{2}_{F},where ОІрқӣҪ\beta is a hyperparameter to adjust the strength of regularization.

Inevitably, the choice training data рқ’ҹрқ’ҹ\mathcal{D} will face sparsity issue for two reasons: 1) labeled nodes are limited in some applications; and 2) only a small portion of the labeled nodes satisfy the criteria of рқ’ҹрқ’ҹ\mathcal{D}. To tackle the data sparsity issue, we have two main considerations: 1) the complexity of the binary classifier should be controlled strictly. Towards this end, we devise the choice model as a Support Vector MachineВ (Pedregosa etВ al., 2011) (SVM), since SVM only requires a few samples to serve as the support vectors to make choice. 2) the inputs of the binary classifier should free from the number of classes LрқҗҝL, which can be large in some applications. To this end, we distill low dimensional and representative factors from the two predictions (i.e., рқ’ҡ^^рқ’ҡ\hat{\bm{y}} and рқ’ҡ^ssuperscript^рқ’ҡрқ‘ \hat{\bm{y}}^{s}) and the causal effect рқ’Ҷрқ’Ҷ\bm{e}) to serve as the inputs of the choice model, which is detailed in SectionВ 3.3.

To summarize, as compared to conventional GCN inference, the proposed CGI has two main differences:вҖўIn addition to the original prediction, CGI calls for causal intervention to further make a post-intervention prediction.вҖўCGI makes choice between the original prediction and post-intervention prediction with a choice model.

Below summarizes the slight change of GCNвҖҷs training and inference schema to apply the proposed CGI:

To reduce the complexity of the choice model, we distill three types of factors as the input: causal uncertainty, prediction confidence, category transition.

According to the homophily assumptionВ (McPhersonetВ al., 2001), the neighbors should not largely changes the prediction of the target node. Therefore, the target node may face the local structure discrepancy issue if the causal effect рқ’Ҷрқ’Ҷ\bm{e} of the neighbors has large variance. That is, the causal effect exhibits high uncertainty w.r.t. different reference values.Inspired by the Monte Carlo uncertainty estimation, we resort to the variance of рқ’Ҷрқ’Ҷ\bm{e} to describe the causal uncertainty, which is formulated as:(11)рқ’—=vвҖӢaвҖӢrвҖӢ({fвҖӢ(рқ’ҷ,рқ’©вҖӢ(рқ’ҷ)k|Оё^)|kвүӨK}),рқ’—рқ‘Јрқ‘Һрқ‘ҹconditionalрқ‘“рқ’ҷconditionalрқ’©subscriptрқ’ҷрқ‘ҳ^рқңғрқ‘ҳрқҗҫ\displaystyle\bm{v}=var(\{f(\bm{x},\mathcal{N}(\bm{x})_{k}|\hat{\theta})|k\leq K\}),where рқ’©вҖӢ(рқ’ҷ)kвҠӮрқ’©вҖӢ(рқ’ҷ)рқ’©subscriptрқ’ҷрқ‘ҳрқ’©рқ’ҷ\mathcal{N}(\bm{x})_{k}\subset\mathcal{N}(\bm{x}), vвҖӢaвҖӢrвҖӢ(вӢ…)рқ‘Јрқ‘Һрқ‘ҹвӢ…var(\cdot) is an element-wise operation that calculates the variance on each class over the KрқҗҫK samples, and рқ’—вҲҲв„ӣLрқ’—superscriptв„ӣрқҗҝ\bm{v}\in\mathcal{R}^{L} denotes class-wise variance. In particular, we perform KрқҗҫK times of causal intervention with N=рқ’©вҖӢ(рқ’ҷ)kрқ‘Ғрқ’©subscriptрқ’ҷрқ‘ҳN=\mathcal{N}(\bm{x})_{k} and then calculate the variance of the corresponding KрқҗҫK causal effects.If an entry of рқ’—рқ’—\bm{v} exhibits a large value, it reflects that minor changes on the subgraph structure can cause large changes on the prediction probability over the corresponds class. According to the original classification z^^рқ‘§\hat{z}, we select the z^^рқ‘§\hat{z}-th entry of рқ’—рқ’—\bm{v} as a representative of the Monte Carlo causal effect uncertainty, which is termed as graph_var.In practice, we calculate the post-intervention predictions by repeating KрқҗҫK times of GCN inference with edge dropoutВ (RongetВ al., 2019) applied, i.e., each edge has a probability П„рқңҸ\tau to be removed.

There has been a surge of attention on using model predictions such as model distillationВ (HintonetВ al., 2015) and self-supervised learningВ (Chen etВ al., 2020b). The intuition is that a larger probability indicates higher confidence on the classification. As such, a factor of prediction reliability is the prediction confidence, i.e., trusting the prediction with higher confidence. Formally, we calculate two factors: self_conf (рқ’ҡ^z^subscript^рқ’ҡ^рқ‘§\hat{\bm{y}}_{\hat{z}}) and neighbor_conf (рқ’ҡ^z^sssubscriptsuperscript^рқ’ҡрқ‘ superscript^рқ‘§рқ‘ \hat{\bm{y}}^{s}_{\hat{z}^{s}}), respectively.

The distribution of edges over categories is not uniform w.r.t. : the ratio of intra-category connection and inter-category connection. Over the labeled nodes, we can calculate the distribution and form a category transition matrix Tрқ‘ҮT where Ti,jsubscriptрқ‘Үрқ‘–рқ‘—T_{i,j} is the ratio of edges between category iрқ‘–i and jрқ‘—j to the edges connect category iрқ‘–i. Figure 3 illustratesan example on the OGB-arXiv dataset (raw normalized). We can see that the probability of intro-category connection (diagonal entries) varies in a large range ([0.19, 0.64]). The distribution of inter-category probability is also skewed. Intuitively, such probabilities can be clues for choosing the correct prediction, e.g., рқ’ҡ^^рқ’ҡ\hat{\bm{y}} might be trustworthy if Tz^,z^subscriptрқ‘Ү^рқ‘§^рқ‘§T_{\hat{z},\hat{z}} is high. To this end, we calculate four factors: self_self (Tz^,z^subscriptрқ‘Ү^рқ‘§^рқ‘§T_{\hat{z},\hat{z}}), neighbor_neighbor (Tz^s,z^ssubscriptрқ‘Үsuperscript^рқ‘§рқ‘ superscript^рқ‘§рқ‘ T_{\hat{z}^{s},\hat{z}^{s}}), self_neighbor (Tz^,z^ssubscriptрқ‘Ү^рқ‘§superscript^рқ‘§рқ‘ T_{\hat{z},\hat{z}^{s}}), and neighbor_self (Tz^s,z^subscriptрқ‘Үsuperscript^рқ‘§рқ‘ ^рқ‘§T_{\hat{z}^{s},\hat{z}}).

We conduct experiments on seven node classification datasets to answer the following research questions:RQ1: How effective is the proposed CGI model to resolve the local structure discrepancy issue? RQ2: To what extend the proposed CGI facilitates node classification under different problem settings? RQ3: How do the distilled factors influence the effectiveness of the proposed CGI?

For the full-supervised settings, we use the widely used benchmark dataset of citation network, OGB-arXivВ (Hu etВ al., 2020a), which represents papers and their citation relations as nodes and edges, respectively. Each node has 128 features generated by averaging the embeddings of words in its title and abstract, where the embeddings are learned by the skip-gram modelВ (Mikolov etВ al., 2013). Considering that the such old-fashioned text feature may not be representative, we replace the node features with a 768-dimensional vector extracted by feeding the title and abstract into RoBERTaВ (Liu etВ al., 2019) (12-layer), where the representation of [CLS] token at the second last layer is selected.

For the semi-supervised settings, we adopt three widely used citation networks, Cora, Citeseer, and Pubmed, and select the 20-shot data split released byВ (Kipf and Welling, 2017), where 500 and 1000 nodes are selected as validation and testing, 20 nodes from each category are labeled for training. Apart from the real-world graphs, we further created three synthetic ones based on Citeseer by intentionally adding cross-category edges on 50% randomly selected nodes, which leads to local structure discrepancy between the poisoned nodes and the unaffected ones. Note that more cross-category edges lead to stronger discrepancy, making adaptive locality more critical for GCN models. In particular, according to the number of edges in the original Citeseer, we add 10%, 30%, and 50% of cross-category edges, constructing Citeseer(10%), Citeseer(30%), and Citeseer(50%). Note that the data split and node features are unchanged.

To justify the proposed CGI, we compare it with the representative GCN, including GraphSAGEВ (HamiltonetВ al., 2017), GCNВ (Kipf and Welling, 2017), GATВ (VeliДҚkoviДҮ etВ al., 2018), JKNetВ (Xu etВ al., 2018), DAGNNВ (Liu etВ al., 2020), and APPNPВ (Klicpera etВ al., 2019), which adopts normal inference. Apart from GCN models, we also test MLP, which discard the graph structure and treat node classification as normal text classification. Lastly, as CGI uses two predictions, we include an ensemble baseline which averages the prediction of APPNP and MLP.For these models, we use the implementations on the OGB leaderboard222https://ogb.stanford.edu/docs/leader_nodeprop/#ogbn-arxiv.. If necessary, e.g., under the transductive full-supervised setting, the hyper-parameters are tuned according to the settings in the original paper of the model. For the proposed CGI, we equipped the SVM with RBF kernel333https://scikit-learn.org/stable/modules/svm.html. and apply CGI to APPNP. For the SVM, we tune two hyper-parameters cрқ‘җc and Оірқӣҫ\gamma through 5-fold cross-validation, i.e., splitting the nodes in validation into 5 folds. In addition, for the MCE that estimates graph uncertainty, we set the number of repeats and the edge dropout ratio П„рқңҸ\tau as 50 and 0.15.

According to the format of the convolution operations, existing GCN models can be divided into two categories: spatial GCN and spectral GCNВ (ZhangetВ al., 2020a). Spectral GCN is defined as performing convolution operations in the Fourier domain with spectral node representationsВ (BrunaetВ al., 2014; Defferrard etВ al., 2016; Kipf and Welling, 2017; LiaoetВ al., 2019; XuetВ al., 2019c). For instance, Bruna et al. (BrunaetВ al., 2014) perform convolution over the eigenvectors of graph Laplacian which are treated as the Fourier basis.Due to the high computational cost of the eigen-decomposition, a line of spectral GCN research has been focused on accelerating the eigen-decomposition with different approximation techniquesВ (Defferrard etВ al., 2016; Kipf and Welling, 2017; LiaoetВ al., 2019; XuetВ al., 2019c).However, applying such spectral GCN models on large graphs still raises unaffordable memory cost, which hinders the their practical research.

To some extent, the attention on GCN research has been largely dedicated on the spatial GCN, which performs convolution operations directly over the graph structure by aggregating the features from spatially close neighbors to a target nodeВ (Atwood andTowsley, 2016; HamiltonetВ al., 2017; Kipf and Welling, 2017; VeliДҚkoviДҮ etВ al., 2018; Xinyi and Chen, 2019; VeliДҚkoviДҮ etВ al., 2019; XuetВ al., 2019b).This line of research mainly focuses on the development of the neighbor aggregation operation. For instance,Kipf and WellingВ (Kipf and Welling, 2017) propose to use a linear aggregator (i.e., weighted sum) that uses the reverse of node degree as the coefficient.In addition to aggregating information from directly connected neighbors, augmented aggregators also account for multi-hop neighborsВ (XuetВ al., 2019a; JiaetВ al., 2020).Moreover, non-linear aggregators are also employed in spatial GCNs such as capsuleВ (VeliДҚkoviДҮ etВ al., 2019) and Long Short-Term Memory (LSTM)В (HamiltonetВ al., 2017).Besides, the general spatial GCN designed for simple graphs is extended to graphs with heterogeneous nodesВ (WangetВ al., 2019b) and temporal structureВ (Park and Neville, 2019).Beyond model design, there are also studies on the model capability analysisВ (XuetВ al., 2019b), model explanationВ (Ying etВ al., 2019), and training schemaВ (Hu etВ al., 2020b).

However, most of the existing studies focus on the training stage and blindly adopt the one-pass forward propagation for GCN inference. This work is in an orthogonal direction, which improve the inference performance with an causal inference mechanism so as to better solve the local structure discrepancy issue. Moreover, to the best of our knowledge, this work is the first to introduce the causal intervention and causal uncertainty into GCN inference.

Amongst the GCN research, a surge of attention has been especially dedicated to solving the over-smoothing issueВ (Li etВ al., 2018). Adaptive locality has become the promising solution to alleviate the over-smoothing issue, which is typically achieved by the attention mechanismВ (VeliДҚkoviДҮ etВ al., 2018; Xu etВ al., 2018; WangetВ al., 2019b; YunetВ al., 2019; WangetВ al., 2020, 2019c; Chen etВ al., 2019a; ChenetВ al., 2020d) or residual connectionВ (Kipf and Welling, 2017; ChenetВ al., 2020e; LietВ al., 2020). Along the line of research on attention design, integrating context information into the calculation of attention weight is one of the most popular techniques. For instance, Wang et al. В (WangetВ al., 2020) treats the neighbors at different hops as augmentation of attention inputs. Moreover, to alleviate the issue of lacking direct supervision, Wang et al. В (WangetВ al., 2019c) introduce additional constraints to facilitate attention learning.Similar as Convolutional Neural Networks, residual connection has also been introduced to original design of GCNВ (Kipf and Welling, 2017), which connects each layer to the output directly. In addition to the vanilla residual connection, the revised versions are also introduced such as the pre-activation residualВ (LietВ al., 2020) and initial residualВ (ChenetВ al., 2020e). Besides, the concept of inception module is also introduced to GCN modelВ (Kazi etВ al., 2019), which incorporates graph convolutions with different receptive fields. For the existing methods, the adaptive locality mechanism is fixed once the GCN model is trained. Instead, this work explores adaptive locality during model inference, which is in an orthogonal direction.

A surge of attention is being dedicated to incorporating causality into the ML schemaВ (YangetВ al., 2021; Zhang etВ al., 2020c; NanetВ al., 2021; ZhangetВ al., 2021; YueetВ al., 2020; HuetВ al., 2021). A line of research focuses on enhancing the inference stage of ML model from the cause-effect viewВ (YueetВ al., 2021; WangetВ al., 2021; NiuetВ al., 2021; TangetВ al., 2020a). This work differs from them for two reasons: 1) none of the existing work studies GCN; and 2) we learn a choice model to make final prediction from causal intervention results instead of performing a heuristic causal inference for making final prediction.

This paper revealed that learning an additional model component such as graph attention is insufficient for addressing the local structure discrepancy issue of GCN models. Beyond model training, we explored the potential of empowering the GCNs with adaptive locality ability during the inference. In particular, we proposed an causal inference mechanism, which leverages the theory of causal intervention, generating a post intervention prediction when the model only trusts own features, and makes choice between the post intervention prediction and original prediction. A set of factors are identified to characterize the predictions and taken as inputs for choosing the final prediction. Especially, we proposed the Monte Carlo estimation of neighborsвҖҷ causal effect. Under three common settings for node classification, we conducted extensive experiments on seven datasets, justifying the effectiveness of the proposed CGI.
