In recent years, Graph Neural Networks (GNNs) have demonstrated great practical performance for analyzing graph-structured data (yu2020tagnn, ). GNNs usually consist of different components such as attention, aggregation, activation functions and so on. In traditional GNNs, the selection of components is a tough problem and determined by manual-based experience, which is difficult to achieve optimal results. Recently, neural architecture search (NAS) has attracted increasing research interests (elsken2019neural, ).NAS aims to explore the optimal neural architecture from search space for maximizing model performance. As shown in Fig. 1, there are a lot of components of GNNs architecture, such as attention, aggregation and activation function, which form the search space of graph neural networks. Architecture search algorithm samples a combination of components from search space S𝑆S such as {g​a​t,s​u​m,t​a​n​h,…}𝑔𝑎𝑡𝑠𝑢𝑚𝑡𝑎𝑛ℎ…\{gat,sum,tanh,...\} in the example as a GNNs architecture s𝑠s. The architecture s𝑠s is further evaluated by performance estimation strategy. The search algorithm uses the estimated performance to generate new and better GNNs architecture from search space S𝑆S.Recently, some works have focused on the problem of graph neural architecture search. GraphNAS (gao2020graph, ) uses a recurrent network to generate fixed-length strings that describe the architectures of graph neural networks, and trains the recurrent network with policy gradient to maximize the expected accuracy of the generated architectures on a validation data set. Gene-GNN (shi2020evolutionary, ) is proposed through the evolution approach to identify both the GNNs architecture and training hyper-parameter in two search spaces.

However, the main challenge of graph neural architecture search is the efficiency since the search space is explosively related to the graph neural network structure. For example, there is about 108superscript10810^{8} possibility of combinations for two layers GNNs architecture (gao2020graph, ). Exploring all possible GNNs architectures in the huge search space is too time-consuming or impossible for big graph data.

In this paper, we propose a parallel graph neural architecture search framework (G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS) that can speed up the search with high accuracy simultaneously.In the G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS, we design sharing-based evolution learning to enhance the diversity of population. For the evolutionary search, we utilize information entropy to assess the mutation selection probability of architecture component, which can reduce exploration effectively in the huge search space for achieving good GNNs architecture. The main contributions of this paper are summarized as follows:•To the best of our knowledge, we make the first attempt to parallel search framework for graph neural architecture search.•We propose a novel neural architecture search framework G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS to find the optimal GNNs architecture automatically. In the G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS, we design sharing-based evolution learning and adopt architecture information entropy to restrict search direction, which can improve the search efficiency without losing the accuracy .•The experiments shows that G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS outperforms state-of-art models in both efficiency and accuracy on real-world datasets.

With the analyzing of architecture features of GNNs, we find that different architecture component value has different frequency of occurrence in the GNNs architectures that can get good performance on given dataset.Inspired by the theory that association algorithm mines frequent itemset(hahsler2007introduction, ), in order to restrict search direction in the huge search space for finding good GNNs architecture faster, we use architecture information entropy to dynamically evaluate the mutation selection probability. Next, architecture components are selected based on the mutation selection probability to perform random mutation operation.

Parallel Initialization.We use multiple workers to explore different areas in the huge search space simultaneously for speed up search process.Using N𝑁N workers to randomly explore N×M𝑁𝑀N\times M GNNs architectures as the initial population in parallel, estimating the validation accuracy as the fitness on the dataset Dv​a​lsubscript𝐷𝑣𝑎𝑙D_{val} for GNNs architectures of initial population.

Sharing Population.The sharing mechanism is designed to make each worker can utilize the GNNs architecture generated by other worker, it can improve diversity of population for evolution search.Ranking GNNs architectures based on fitness for initial population, and extracting top n𝑛n GNNs architectures as the sharing population.

The architecture mutation selection probability vector P→→𝑃\vec{P} is a soft constraint strategy for evolution search, which can restrict search direction on the region near the GNNs architectures that have a good performance and reserve probability of exploring other areas at the same time.

Parallel Search.Using wheel selection strategy to sample k𝑘k GNNs architectures from sharing population as the parents for each worker. Selecting m𝑚m architecture components to perform random mutation based on architecture mutation selection probability vector P→→𝑃\vec{P}. Each worker will generate k𝑘k child architectures and evaluate fitness for these child architectures on the dataset Dv​a​lsubscript𝐷𝑣𝑎𝑙D_{val}.

Then, putting child architecture into sharing population if the fitness of child architecture is greater than threshold F𝐹F, which F𝐹F is the average fitness of top n𝑛n GNNs architectures in sharing population.

In this section, we conduct extensive experiments on three benchmark datasets to evaluate the effectiveness of G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS for node classification task.

In this work, we use three benchmark citation networks including Citeseer, Cora and Pubmed (sen2008collective, ) which nodes represent documents and edges denote citation links.Dataset characteristics of statistics are summarized in Table 2.

In this study, we compare GNNs architecture identified by our model against baselines including the state-of-art handcrafted architectures and NAS methods for node classification task.Handcrafted architectures are included: Chebyshev (defferrard2016convolutional, ), GCN (kipf2016semi, ), GraphSAGE (hamilton2017inductive, ), GAT (velivckovic2017graph, ), LGCN (gao2018large, ) and LC-GCN/GAT (xu2020label, ).NAS methods are involved: Auto-GNN (zhou2019auto, ), GraphNAS (gao2020graph, ) and Gene-GNN (shi2020evolutionary, ).

The GNNs architecture designed by G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS are implemented by the Pytorch-Geometric library(fey2019fast, ).

Hyper-parameters of search algorithm: The parallel workers N𝑁N as 4, the number of layer l𝑙l is fixed to 2, the size of random initial population M𝑀M as 100 for each worker, the number of sharing population n𝑛n and the number of parents k𝑘k as 20, the number of mutation m𝑚m is set to 1, 2, 3, 4 for each worker separately, finally the s​e​a​r​c​h​_​e​p​o​c​h𝑠𝑒𝑎𝑟𝑐ℎ_𝑒𝑝𝑜𝑐ℎsearch\_epoch as 20.

Hyper-parameters of GNNs: For training one GNNs architecture designed by G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS, the training e​p​o​c​h𝑒𝑝𝑜𝑐ℎepoch as 300, and we used the L​2𝐿2L2 regularization with λ𝜆\lambda = 0.0005, the learning rate r𝑟r = 0.005, and dropout probability p𝑝p = 0.6, we utilized A​d​a​m𝐴𝑑𝑎𝑚Adam S​G​D𝑆𝐺𝐷SGD to minimize cross-entropy loss function.

To validate the performance of our model, we compare the GNNs models discovered by G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS with handcrafted models and those designed by other search approaches.Test accuracy on the node classification task is summarized in Table 3, where the best performance is highlighted in bold.

Considering the deviation of testing accuracy, the test performance of G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS is averaged via randomly initializing the optimal GNNs architecture 20 times, and those of handcrafted architectures as well as NAS methods are reported directly from their works.In general, the NAS methods can achieve better results than handcrafted methods, which demonstrated that it is effective for NAS methods to design good GNNs architecture for the given graph data.Especially, the G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS get the best performance on all three benchmark datasets.The reason is that the handcrafted models need a lot of tuning of GNNs architecture parameters by manual method for different graph datasets, which is hard to obtain an optimal model.On the contrary, the NAS methods can automatically search the GNNs architecture with the ability that makes the performance of GNNs model better for given graph datasets, which can gradually optimize the GNNs model performance with litter human intervention.

To further investigate the effectiveness of the G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS, we compare the progression of top-10 averaged validation performance of GraphNAS, random search and G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS.

For each search approach, 2,000 GNNs architectures are explored in the same search space.As shown in Figure 3, G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS is more efficient to find the well-performed GNNs architectures during the search process.Because G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS can search the GNNs architecture in different areas simultaneously in the huge search space based on parallel ability.Then, the sharing mechanism can enhance the diversity of population, which is beneficial to improve the search efficiency for evolution algorithm.Finally, G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS utilizes architecture information entropy to dynamically compute mutation selection probability that can restrict search direction effectively for achieving good GNNs architecture faster.

In this paper, we propose a parallel architecture search framework G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS for graph neural networks. G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS utilizes parallel ability and restricts search direction effectively to speed up search process and achieve optimal GNNs architecture automatically. Experiment results on the three benchmark datasets show that G​r​a​p​h​P​A​S𝐺𝑟𝑎𝑝ℎ𝑃𝐴𝑆GraphPAS can obtain GNNs architecture that outperforms the state-of-art method with efficiency and accuracy.
