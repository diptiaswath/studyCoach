Weighted finite state automata (WFSA) have been a vital tool in building automatic speech recognition (ASR) systems for both model training & inference.Common neural ASR models (CTC [1], RNN-T [2]) exhibit a close connection to the WFSA formalism, and there has been a renewed interest in casting these models into WFSAs, e.g. frameworks such as k2 [3], GTN [4], and PyTorch-Struct [5].More recently, [6] demonstrates globally normalized sequence models built within the WFSA formalism can greatly reduce the word error rate of streaming ASR models.

Motivated by the need for efficient yet flexible and easy-to-use sequence modelling tools for implementing the aforementioned methods, especially the globally normalized models such as [6], we developed LAST, a lattice-based speech transducer library in JAX [7].LAST provides a simple interface for operations over a recognition lattice, and is fully compatible with core features in JAX such as automatic differentiation and function transformations.Being written in pure JAX allows LAST to run efficiently on CPUs, GPUs, and TPUs without any code modification, whereas the existing WFSA based sequence modelling toolkits [4, 3] employ handwritten kernels targeting a limited set of hardware.

In this paper, we introduce the design of LAST, discuss the technical challenges that drive crucial design choices, and present benchmark results.LAST is open source software under the Apache License 2.0.The complete code can be found at https://github.com/google-research/last.

For example, the computation of the RNN-T loss [2] can be broken down into two WFSA operations:(1) Intersecting the (infinite) complete recognition lattice with the reference output string (Figure 1 of [2]);(2) Shortest distance on the intersection.Another example is the computation of the globally normalized loss in [6]:(1) Shortest distance on the complete recognition lattice to obtain the denominator;(2) Similar to RNN-T, intersection with the reference and then shortest distance to obtain the numerator.

During training, we differentiate through shortest distance & intersection to obtain the gradients of the training loss with respect to the arc weights.

We chose to implement LAST in JAX because of the performance, ease of use, and flexibility JAX offers.JAX [7] is a high-performance machine learning research toolkit that has seen a growing popularity.JAX programs written in the NumPy API execute efficiently on CPUs, GPUs and TPUs.Similar to [10], the vector-Jacobian-product function needed for (reverse mode) automatic differentiation [11, 12] can be either automatically derived for any function expressed in differentiable JAX primitives, or manually defined.

The GNAT model outlined in [6] is a modular theoretical framework encompassing all the common neural speech recognition models.[6] reports that the globally normalized GNAT model greatly reduces the word error rate of streaming ASR.The primary goal of LAST is to replicate GNAT.Therefore, the top level API of LAST closely follows components of the GNAT model.GNAT breaks down modelling choices into 3 components:The context dependency is an unweighted deterministic finite automaton that accepts Σ∗superscriptΣ\Sigma^{*}.The states in a context dependency encode the output history.The alignment lattice for a given input length T𝑇T is an acyclic unweighted finite automaton.The states in an alignment lattice encode the correspondence between input frames and output alignment labels.The weight function is a function QT×QC×(Σ∪{ϵ})→𝕂→subscript𝑄𝑇subscript𝑄𝐶Σitalic-ϵ𝕂Q_{T}\times Q_{C}\times(\Sigma\cup\{\epsilon\})\rightarrow\mathbb{K}.The intersection of the context dependency and the alignment lattice defines the topology of the recognition lattice, whereas the weight function defines the weights of each arc.Figures 1(a), 1(b), and 1(c) are a concrete example of these components from [6].

The core of the LAST library, class RecognitionLattice is a flax.linen module [15] that combines these 3 components to produce the actual states and weights for any recognition lattice induced from a sequence of input frames.The shortest distance of either the entire RecognitionLattice, or its intersection with an output string, can then be obtained through API methods for training locally or globally normalized models.A low level state/arc-based API is also provided for applications such as beam search decoding.

While class RecognitionLattice provides efficient high level operations through a simple interface by hiding implementation details, the modelling components are instead designed as easy to extend interfaces,•In addition to the FullNGram context dependency described in [6], there is also a NextStateTable context dependency, enabling the use of any context dependency topology with a look-up table.•Both FrameDependent and FrameLabelDependent alignment lattices are available.•Weight functions can be implemented as the combination of a WeightFnCacher that performs precomputation of static data (such as context state embeddings), and a WeightFn that actually produces arc weights.

WFSAs in LAST are not just defined with data (i.e. arrays of state/arc tables) but also with code that describes higher level operations.This is a key difference of LAST from prior art such as k2 [3].For example, ContextDependency provides ForwardReduce and BackwardBroadcast, simple yet crucial primitives that RecognitionLattice uses to produce a highly vectorized forward-backward implementation (Section LABEL:sec:random-access).Such higher level APIs are designed to take advantage of special structures in certain WFSAs to achieve higher efficiency with minimum interface complexity.

The main challenge in scaling up differentiable WFSA algorithms for modern accelerator architectures is memory: the memory is more limited in size compared to CPU DRAM, and memory access can be much slower than compute.Efficient memory use is thus behind many design details in LAST.

The memory usage of our implementation in k2 shows that on-the-fly arc weight computation can bring about significant memory savings.k2’s training speed is faster than LAST because computing all arc weights at once affords more opportunity for parallelization: we compute arc weights 1 frame at a time in LAST, but 16 frames at a time in k2.Had we also computed arc weights for 1 or 2 frames at a time in k2, the training time would increase to 4.76s or 2.59s respectively.

We presented the LAST library and discussed challenges in scaling up WFSA algorithms in an automatic differentiation framework.These challenges led us to design LAST differently from prior art.Our generally applicable techniques to counter these challenges proved effective as shown in the benchmark results.
