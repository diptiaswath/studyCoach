Conditional gradient methods (CGM), such as Frank-Wolfe and its variants, are well-known optimization approaches that have been extensively used in modern machine learning. For example, CGM has been applied to kernel methods [BLJO12, TLR21], structural learning [LJJSP13] and online learning [FGM17, LHY+21, HK12].

Recent years have witnessed the success of large-scale machine learning models trained on vast amounts of data. In this learning paradigm, the computational overhead of most successful models is dominated by the optimization process [DCLT18, BMR+20]. Therefore, reducing the running time of the optimization algorithm is of practical importance. The total running time in optimization can be decomposed into two components: (1) the number of iterations towards convergence, (2) the cost spent in each iteration. Reducing the number of iterations requires a better understanding of the geometric proprieties of the problem at hand and the invention of better potential functions to analyze the progress of the algorithm [Jag13, GH15, RSPS16, RHS+16, AZHHL17, LZC+19, WZD+20]. Reducing the cost spent per iteration usually boils down to designing problem-specific discrete data-structures. In the last few years, we have seena remarkable growth of using data-structures to reduce iteration cost [CLS19, LSZ19, BLSS20, DLY21, Bra20, BLN+20, JKL+20, JSWZ21, BLL+21, SY21, BPSW21, SYZ21, HJS+21, SZZ21].

A well-known strategy in optimization, with CGM, is to perform a greedy search over the weight vectors [Jag13, GH15, AZHHL17, LSZ19, FW+56] or training samples [LDH+17, LDL+18] in each iteration. In this situation, the cost spent in each iteration is linear in the number of parameters. In practical machine learning, recent works [CXS19, CMF+20, CLP+21, DMZS21, XCL+21] formulate this linear cost in iterative algorithms as an approximate maximum inner product search problem (𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP}). They speed up the amortized cost per iteration via efficient data-structures from recent advances in approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} [SL14, SL15b, SL15a, NS15, GKCS16, YHLD17, MB18, ZTXL19, TZXL19, GSL+20]. In approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structures, locality sensitive hashing (𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH}) achieves promising performance with efficient random projection based preprocessing strategies [SL14, SL15b, SL15a, NS15]. Such techniques are widely used in practice for cost reduction in optimization. [CXS19] proposes an 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} based gradient sampling approach that reduces the total empirical running time of the adaptive gradient descent. [CMF+20] formulates the forward propagation of deep neural network as a 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} problem and uses 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} to select a subset of neurons for backpropagation. Therefore, the total running time of neural network training could be reduced to sublinear in the number of neurons. [DMZS21] extends this idea with system-level design for further acceleration, and [CLP+21] modifies the 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} with learning and achieves promising acceleration in attention-based language models. [XCL+21] formulates the greedy step in iterative machine teaching (IMT) as a 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} problem and scale IMT to large datasets with 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH}. [SYZ21] shows how to use the exact 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structure to speedup the training of one-hidden layer over-parameterized ReLU neural network.

Despite the practical success of cost-efficient iterative algorithms with approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structure, the theoretical analysis of its combination with CGM is not well-understood. In this paper, we focus on this combination and target answering the following questions: (1) how to transform the iteration step of CGM algorithms into an approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} problem? (2) how does the approximate error in 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} affect CGM in the total number of iterations towards convergence? (3) how to adapt approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data structure for iterative CGM algorithms?

We propose a theoretical formulation for combining approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} and convergence guarantees of CGM. In particular, we start with the popular Frank-Wolfe algorithm over the convex hull where the direction search in each iteration is a 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} problem. Next, we propose a sublinear iteration cost Frank-Wolfe algorithm using 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} type 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structures. We then analyze the trade-off of approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} and its effect on the number of iterations needed by CGM to converge. We show that the approximation error caused by 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} only leads to a constant multiplicative factor increase in the number of iterations. As a result, we retain the sub-linearly of 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH}, with respect to the number of parameters, and at the same time retain the same asymptotic convergence as CGMs.

We summarize our complete contributions as follows.

•We give the first theoretical CGM formulation that achieves provable sublinear time cost per iteration. We also extend this result into Frank-Wolfe algorithm, Herding algorithm, and policy gradient method.•We propose a pair of efficient transformations that formulates the direction search in Frank-Wolfe algorithm as a projected approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} problem.•We present the theoretical results that the proposed sublinear Frank-Wolfe algorithm asymptotically preserves the same order in the number of iterations towards convergence. Furthermore, we analyze the trade-offs between the saving in iteration cost and the increase in the number of iterations to accelerate total running time.•We identify the problems of 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} type approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} for cost reduction in the popular CGM methods and propose corresponding solutions.

Maximum Inner Product Search (𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP}) is a fundamental problem with applications in machine learning. Given a query x∈ℝd𝑥superscriptℝ𝑑x\in\mathbb{R}^{d} and an n𝑛n-vector dataset Y⊂ℝd𝑌superscriptℝ𝑑Y\subset\mathbb{R}^{d}, 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} targets at searching for y∈Y𝑦𝑌y\in Y that maximizes the inner product x⊤​ysuperscript𝑥top𝑦x^{\top}y. The naive 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} solution takes O​(d​n)𝑂𝑑𝑛O(dn) by comparing x𝑥x with each y∈Y𝑦𝑌y\in Y. To accelerate this procedure, various algorithms are proposed to reduce the running time of 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} [SL14, SL15b, NS15, SL15a, GKCS16, YHLD17, YLD+18, DYH19, MB18, ZLW+18, ZTXL19, TZXL19, GSL+20]. We could categorize the 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} approaches into two categories: reduction methods and non-reduction methods. The reduction methods use transformations that transform approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} into approximate nearest neighbor search (𝖠𝖭𝖭𝖠𝖭𝖭\mathsf{ANN}) and solve it with 𝖠𝖭𝖭𝖠𝖭𝖭\mathsf{ANN} data-structures. One of the popular data-structure is locality sensitive hashing [IM98, DIIM04].

Here we define the 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} functions for euclidean distance. 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} functions could be used for search in cosine [Cha02, LL19] or Jaccard similarity [LLZ19, LL21]. [SL14] first shows that 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} could be solved by ℓ2subscriptℓ2\ell_{2} 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} and asymmetric transformations. After that, [SL15b, NS15, SL15a, YLD+18] propose a series of methods to solve 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} via 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} functions for other distance measures. Besides 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH}, graph-based 𝖠𝖭𝖭𝖠𝖭𝖭\mathsf{ANN} approaches [YHLD17] could also be used after reduction.

On the other hand, the non-reduction method directly builds data-structures for approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP}. [GKCS16, GSL+20] use quantization to approximate the inner product distance and build codebooks for efficient approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP}. [YHLD17, DYH19] propose a greedy algorithm for approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} under computation budgets. [MB18, ZTXL19, TZXL19] directly construct navigable graphs that achieve state-of-the-art empirical performance.

Recently, there is a remarkable growth in applying data-structures for machine learning [BIW19, ZNV+20, BDI+20]. Following the paradigm, approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structures have been applied to overcome the efficiency bottleneck of various machine learning algorithms. [YHLD17] formulates the inference of a neural network with a wide output layer as a 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} problem and uses a graph-based approach to reduce the inference time. In the same inference task, [LXJ+20] proposes a learnable 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} data-structure that further improves the inference efficiency with less energy consumption. In neural network training, [CMF+20, CLP+21, DMZS21] uses approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} to retrieve interested neurons for backpropagation. In this way, the computation overhead of gradient updates in neural networks could be reduced. In linear regression and classification models, [CXS19] uses approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structures to retrieve the samples with large gradient norm and perform standard gradient descent, which improves the total running time for stochastic gradient descent. [XCL+21] proposes a scalable machine teaching algorithm that enables iterative teaching in large-scale datasets. In bandit problem, [YRS+21] proposes an LSH based algorithm that solves linear bandits problem with sublinear time complexity.

Despite the promising empirical results, there is little theoretical analysis on approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} for machine learning. We summarize the major reasons as: (1) Besides 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH}, the other approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structures do not provide theoretical guarantees on time and space complexity. (2) Current approaches treat data-structures and learning dynamics separately. There is no joint analysis on the effect of approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} for machine learning.

Previous literature focuses on reducing the number of iterations for Frank-Wolfe algorithm over specific domains such as ℓpsubscriptℓ𝑝{\ell}_{p} balls [Jag13, GH15, AZHHL17, LZC+19]. In optimization, the gradient computed in every iteration is not independent of each other. This would generate a problem for 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structures. If we use a vector containing the gradients as a query for 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structures, the query failure probability in each iteration is not independent. Therefore, the total failure probability could not be union bounded. As previous 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structures focus on the assumptions that queries are independent, the original failure analysis could not be directly applied.

This work uses a standard query quantization method to handle the adaptive query sequence in optimization. Given the known query space, we quantize it by lattices [GS88]. This quantization is close to the Voronoi diagrams. In this way, each query is located into a cell with a center vector. Next, we perform a query using the center vector in the cell. Therefore, the failure probability of the 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} query sequence is equivalent to the probability that any center vector in the cell fails to retrieve its approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} solution. As the centers of cells are independent, we could union bound this probability. On the other hand, as the maximum diameter of the cell is λ𝜆\lambda, this query quantization would introduce a λ𝜆\lambda additive error in the inner product retrieved. We refer the readers to Appendix G for the detailed quantization approach.

In this work, we show that by 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} based 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structure, the cost for direction search is O​(d​nρ)𝑂𝑑superscript𝑛𝜌O(dn^{\rho}), where ρ∈(0,1)𝜌01\rho\in(0,1). In Section D.2 of the supplementary material, we show that ρ𝜌\rho is a function of constant c𝑐c and parameter τ𝜏\tau in approximate 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} (see Definition 4.2). Moreover, we also show in Section D.2 that 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} results in only a constant multiplicative factor increase in the number of iterations. Considering the cost per iteration and the number of iterations, we show that when our algorithms stop at the ϵitalic-ϵ\epsilon-optimal solution, 𝖫𝖲𝖧𝖫𝖲𝖧\mathsf{LSH} could achieve acceleration in the overall running time. Therefore, we could set c𝑐c and τ𝜏\tau parameters to balance the accuracy-efficiency trade-off of CGM to achieve the desired running time.

In this work, we present the first Frank-Wolfe algorithms that achieve sublinear linear time cost per iteration. We also extend this result into Herding algorithm and policy gradient methods. We formulate the direction search in Frank-Wolfe algorithm as a projected approximate maximum inner product search problem with a pair of efficient transformations. Then, we use locality sensitive hashing data-structure to reduce the iteration cost into sublinear over the number of possible parameters. Our theoretical analysis shows that the sublinear iteration cost Frank-Wolfe algorithm preserves the same order in the number of iterations towards convergence. Moreover, we analyze and optimize the trade-offs between saving iteration cost and increasing the number of iterations to achieve sublinear total running time. Furthermore, we identify the problems of existing 𝖬𝖺𝗑𝖨𝖯𝖬𝖺𝗑𝖨𝖯\mathsf{MaxIP} data-structures for cost reduction in iterative optimization algorithms and propose the corresponding solutions. We hope this work can be the starting point of future study on sublinear iteration cost algorithms for optimization.
