Online multiclass classification is a ubiquitous problem in machine learning. In this problem, a learning algorithm is presented with a stream of incoming query points and is tasked with assigning each query with a label from a fixed set. After choosing a label, the algorithm is told the true label of the query point. The goal of the algorithm is to learn over time how to label the query points as accurately as possible.

Traditionally, theoretical treatments of this problem are built around the notion of a margin γ𝛾\gamma. This margin represents the extent to which the input points are well-separated from the boundaries between different labels. For example, the analysis of the classic Perceptron algorithm [20] guarantees that it makes at most O​(1/γ2)𝑂1superscript𝛾2O(1/\gamma^{2}) mistakes when performing binary classification, as long as all query points are distance at least γ𝛾\gamma from a hyperplane separating the two classes. More sophisticated analyses and algorithms (relying on e.g. hinge loss) do not necessarily assume the classes are as well separated, but still inherently incorporate a margin γ𝛾\gamma (for example, the hinge loss associated with a point is positive unless it is γ𝛾\gamma-separated).

In this paper, we present an alternative to the traditional margin approaches. Our approach weights each mistake by how ambiguous the classification task is for that point, rather than penalizing all mistakes equally. More precisely, consider a partition of the space of all possible query points into k𝑘k regions Risubscript𝑅𝑖R_{i}, where Risubscript𝑅𝑖R_{i} contains all query points whose true label is i𝑖i. In our formulation assigning a query point q𝑞q a label i𝑖i incurs a loss of ℓ​(q,Ri)ℓ𝑞subscript𝑅𝑖\ell(q,R_{i}), where ℓ​(q,Ri)ℓ𝑞subscript𝑅𝑖\ell(q,R_{i}) should be thought of as the distance needed to move q𝑞q so that it lies in Risubscript𝑅𝑖R_{i} (i.e., for it to be labelled correctly). For example, in the case of a linear classifier, ℓ​(q,Ri)ℓ𝑞subscript𝑅𝑖\ell(q,R_{i}) is zero if q𝑞q is correctly classified, and the distance to the classifier if q𝑞q is incorrectly classified. The goal of the algorithm is to minimize the total loss.

This notion of loss not only measures the rate of errors but also the degree of each error; choosing a wildly inaccurate label is punished more than selecting a label that is ”almost” correct. This fine-grained approach to looking at errors has occurred in other areas of machine learning research as well. For example, the technique of knowledge distillation is based on training a smaller model on the logits produced by a larger model [10]. Hinton et al. explain, “The relative probabilities of incorrect answers tell us a lot about how the cumbersome model tends to generalize. An image of a BMW, for example, may only have a very small chance of being mistaken for a garbage truck, but that mistake is still many times more probable than mistaking it for a carrot.” Rather than leaning on the power of a trained model, our framework differentiates between these different incorrect answers based on the geometry of the problem.

Indeed, our algorithm in Theorem 1.1 is not based off of the Perceptron algorithm or its relatives, but rather off of recently developed algorithms for a problem in online learning known as contextual search. In contextual search, a learner is similarly faced with a stream of incoming query points qtsubscript𝑞𝑡q_{t} and wishes to learn a hidden vector v𝑣v. However, instead of trying to predict the sign of ⟨v,qt⟩𝑣subscript𝑞𝑡\left\langle v,q_{t}\right\rangle, in contextual search the goal is to guess the value of ⟨v,qt⟩𝑣subscript𝑞𝑡\langle v,q_{t}\rangle. In the second case, our mistake bound is tight up to a multiplicative factor of k𝑘k; it is an interesting open question whether it is possible to remove this factor of k𝑘k in our techniques.

We additionally introduce a variant on the mistake bound that we call a robust mistake bound that is defined as follows. Normally, when we have a margin constraint of γ𝛾\gamma, we insist that all query points qtsubscript𝑞𝑡q_{t} are distance at least γ𝛾\gamma away from any separating hyperplane. In our robust model, we remove this constraint, but only count mistakes when the query point qtsubscript𝑞𝑡q_{t} lies at least γ𝛾\gamma away from the separating hyperplane.

Existing algorithms (both the Perceptron and halving algorithms) do not appear to give any non-trivial mistake bounds in the robust model – it is very important for the analysis of these algorithms that every query point is far from the separating hyperplane. On the other hand, it straightforwardly follows from our notion of loss that if we have an O​(R)𝑂𝑅O(R)-loss algorithm for some problem, that algorithm simultaneously achieves an O​(R/γ)𝑂𝑅𝛾O(R/\gamma) robust mistake bound. In particular, we obtain robust mistake bounds of O​((d​log⁡d)/γ)𝑂𝑑𝑑𝛾O((d\log d)/\gamma) and O​((k2​d​log⁡d)/γ)𝑂superscript𝑘2𝑑𝑑𝛾O((k^{2}d\log d)/\gamma) for learning linear classifiers and learning nearest neighbor partitions respectively.

The problem of online binary classification (and specifically, of online learning of a linear classifier) is one of the oldest problems in machine learning. The Perceptron algorithm was invented in [21], and the first mistake bound analysis of the Perceptron algorithm appeared in [20]. Since then, there has been a myriad of research on this problem, some of which is well-surveyed in [19]. Of note, the first mistake bounds for the non-separable case appear in [7]. We are not aware of any work on this problem that investigates the same loss we present in this paper.

Bounds for support vector machines (see [23]) also result in the use of a margin to bound the number of mistakes. Choosing a suitable kernel can help create or improve the margin when viewing the data in the “kernel” space. We use a similar technique in proving bounds for generalized Lpsuperscript𝐿𝑝L^{p} norms. Our idea is to use differently-scaled kernels to produce increasingly accurate approximations. To the best of our knowledge, the technique we present is novel. There are other related techniques in the literature, e.g. [22] attempt to learn the best kernel to improve classification error.

Similarly, there is a wealth of both theoretical and empirical research on multiclass classification. As far back as 1973, researchers were looking at generalizing binary classification algorithms to this setting (see e.g. Kesler’s construction in [6]). One piece of relevant work is [5], which generalizes online binary classification Perceptron-style algorithms to solve online multiclass classification problems – in particular, the multiclass hypotheses they consider are same as the nearest neighbor partitions generated by the inner-product similarity metric (although as with the Perceptron, they only analyze the raw classification error). Since then, this work has been extended in many ways to a variety of settings (e.g. [2, 3, 12]).

Another way of looking at the problem of multiclass classification is that we are learning how to cluster a sequence of input points into k𝑘k pre-existing clusters. Indeed, the nearest neighbor partition with L2superscript𝐿2L^{2} similarity metric gives rise to exactly the same partitioning as a k𝑘k-means clustering. There is an active area of research on learning how to cluster in an online fashion (e.g. [9, 8, 1]). Perhaps most relevantly, [17] studies a setting where one must choose cluster labels for incoming points in an online fashion, and then at the end of the algorithm can choose the k𝑘k centers (the goal being to minimize the total k𝑘k-means loss of this eventual clustering).

The algorithms we develop in this paper are based off of algorithms for contextual search. Contextual search is a problem that originally arose in the online pricing community [4, 15]. The variant of contextual search we reference in this paper (with symmetric, absolute value loss) first appeared in [16]. The algorithms in this paper were later improved in [14] (and it is these improved algorithms that we build off of in this paper).

We will show that we can solve this online learning problem with total loss O​(poly​(d))𝑂poly𝑑O({\rm poly}(d)), independent of the number of rounds T𝑇T in the time-horizon. As mentioned earlier, our primary tool will be existing algorithms for a problem in online learning known as contextual search.

Recall that in contextual search there is a hidden vector v∈Bd𝑣subscript𝐵𝑑v\in B_{d}, and each round we are given a vector qt∈Bdsubscript𝑞𝑡subscript𝐵𝑑q_{t}\in B_{d}. As mentioned earlier in Section 2.2, there exist algorithms for contextual search with O​(d​log⁡d)𝑂𝑑𝑑O(d\log d) total loss. Via a simple reduction, we will show that we can apply these algorithms in our setting.

The general idea behind this reduction is simple. We will run our algorithm 𝒜𝒜\mathcal{A} for contextual search to find the hidden point w=x1−x2𝑤subscript𝑥1subscript𝑥2w=x_{1}-x_{2}. There is one important caveat here: how do we update the algorithm 𝒜𝒜\mathcal{A}? Recall that any contextual search algorithm expects binary feedback each round as to whether its guess gtsubscript𝑔𝑡g_{t} was too high or too low. While in some cases, we can provide 𝒜𝒜\mathcal{A} with accurate feedback, in many cases we cannot: for example, if 𝒜𝒜\mathcal{A} submits a guess gt=2.5subscript𝑔𝑡2.5g_{t}=2.5 for ⟨qt,w⟩subscript𝑞𝑡𝑤\langle q_{t},w\rangle, and all we learn is that ⟨qt,w⟩≥0subscript𝑞𝑡𝑤0\langle q_{t},w\rangle\geq 0, we cannot say with confidence whether 𝒜𝒜\mathcal{A}’s guess was too large or not.
