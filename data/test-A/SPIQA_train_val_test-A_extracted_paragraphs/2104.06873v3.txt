Within discrete optimization, the submodularity property hasbeen shown to be a fundamental and useful property.Intuitively, submodularity captures the idea ofdiminishing returns, where the marginal gain in utility decreasesas the set becomes larger.Submodular objective functions arise in many learning objectives,e.g. interpreting neural networks [14],nonlinear sparse regression [15],among many others (seeIyer et al. Challenges from Big Data.Because of ongoing exponential growth in data size [31, 27] over the past decades,much effort has gone into the design ofalgorithms for submodular optimizationwith low time complexity, e.g. [3, 29, 10, 16, 24].In addition to the time complexity of an algorithm,we also consider the number of oracle queries analgorithm makes or thequery complexity; this information is importantas the function evaluation may be much more expensivethan arithmetic operations.Moreover, much effort has also gone intothe design of memory efficient algorithmsthat do not need to store all of the data.In this context, researchers havestudied streaming algorithms forsubmodular optimization [4, 11, 13, 19, 30, 1, 21, 28].A streaming algorithmtakes a constant number of passesthrough the ground set (preferably a single pass) while stayingwithin a small memory footprintof O​(k​log⁡n)𝑂𝑘𝑛O(k\log n), where k𝑘k is the maximum sizeof a solution and n𝑛n is the size of the ground set.

Randomized vs. Deterministic Algorithms.Especially in the case that f𝑓f is non-monotone, randomizationhas been shown to be very useful in designing approximation algorithmsfor SMCC [12, 18, 2].Deterministic algorithms for SMCC have beenmuch less common. In addition to thetheoretical question of the power of deterministicvs. randomized algorithms, thereare practical disadvantages to randomized algorithms:an approximation ratio that holds only in expectationmay result in a poor solution with constant probability.To obtain the ratio with high probability,O​(log⁡n)𝑂𝑛O(\log n) repetitions of the algorithmare typically required, whichmay be infeasible or undesirable in practice, especiallyin a streaming context.Moreover, the derandomization of algorithms for submodularoptimization has proven difficult, althougha method to derandomize some algorithms at the cost of a polynomialincrease in time complexity wasgiven by Buchbinder and Feldman [6].

The Single-Pass Algorithm of [25].Our streaming algorithm LinearStream may be viewed as a generalization of the algorithm of[25] to non-monotone submodular functions;this generalization is accomplished by maintaining twodisjoint candidate solutions X𝑋X and Y𝑌Ythat compete for elements. The loss due to non-monotonicity can then be bounded using theinequality f​(S∪X)+f​(S∪Y)≥f​(S)𝑓𝑆𝑋𝑓𝑆𝑌𝑓𝑆f(S\cup X)+f(S\cup Y)\geq f(S) for any set S𝑆S, which follows from submodularity, nonnegativity, and the fact that X∩Y=∅𝑋𝑌X\cap Y=\emptyset. This strategy of managing the non-monotonicity has been used in the context of greedyalgorithms previously [24, 20].In addition to the above strategy,it becomes necessary to use an unconstrained maximization algorithm on the candidate sets X,Y𝑋𝑌X,Y; this is needed since deletion of elements from the set may cause the function value to increase due to non-monotonicity. To the best of our knowledge, this is a novel use of unconstrained maximization and requires a new procedure that can update its solution in constant time as discussed above.

The degree of interpolationis controlled by a parameter ℓℓ\ell:If ℓ=1ℓ1\ell=1, InterpolatedGreedyreduces to standard greedy, while if ℓ=kℓ𝑘\ell=k, itcan be shown that our algorithmreduces to RandomGreedy.Since we set ℓℓ\ell to a constant value (2​e/ε2𝑒𝜀2e/\varepsilon),the algorithm canconsidered to be closer to StandardGreedy than RandomGreedy.

Overview of Proof. The main difficulty of theproof is showing that InterlaceGreedy is nearly an ℓℓ\ell-approximation(Theorem 5).Once this is established, the proof is similar tothe RandomGreedy analysis, except with ℓℓ\ell iterationsinstead of k𝑘k. To show InterlaceGreedy is an ℓℓ\ell-approximation,it is necessary to order the optimal solution O𝑂O in acertain way, such that each of the ℓℓ\ell greedyprocedures gets a marginal gain that dominates thegain of any of the next ℓℓ\ell elements of O𝑂O. Ifone of the greedy procedures adds an element of O𝑂Oin the first iteration, this may not be possible;hence, we have to guess which of the first ℓℓ\ellelements, if any, is the best element that intersects with O𝑂Oand give each greedy procedure the opportunityto start with this element. This guessing procedure(ℓ+1ℓ1\ell+1 guesses in total) is responsible forthe low (but constant) probability of successof each iteration, as only one of the guesses canbe correct.

Discussion. On almost all instances, LS+ returned nearly the greedy value(typically ≥95%absentpercent95\geq 95\%, except one instance in Fig. 2(c)) while using <3​nabsent3𝑛<3n oracle queries for CIFAR-10 and ba datasets,and <6​nabsent6𝑛<6n oracle queries for fb datasets.The only algorithm with competitive objective valueto LS+ is AEFNS, whichrequires more than an order of magnitude more queries than LS+.Even without post-processing,the objective value of LS is competitive withFKK on some instances; see Fig. 2(c) and 2(e).The algorithm LRVZ, despite having the besttheoretical performance ratio,returns similar solutions compared to LS+; see Fig. 2(a);moreover, LRVZ isthe least scalable algorithm, often requiring more than1000​n1000𝑛1000n queries of the objective.

Our algorithms LS and LS+ used the fewest oracle queries on every evaluatedinstance of SMCC. Moreover, our algorithms exhibited the best scalingwith the size constraint k𝑘k; LS has no k𝑘k dependence at all.After LS and LS+, the next most scalable algorithm on theevaluated instances is FKK, which scales linearly with k𝑘kand used more than an order ofmagnitude more queries than our algorithms.In addition, FKK consistently returns the lowest objective valueof any algorithm.Finally, the post-processing of LS+ adds only a smallamount of extra queries over LS, as shown inFigs. 2(b),2(d), and 2(f), butresults in a large improvement in objective value.
